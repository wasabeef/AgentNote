## Was du bekommst

Das Dashboard gibt deinem Team eine gemeinsame URL für den Verlauf von Agent Note.

- Offene und gemergte PRs an einem Ort
- Commits, Prompts und Attribution ohne jede PR einzeln zu öffnen
- Eine gemeinsame GitHub Pages URL, normalerweise unter `/dashboard/`

Nutze es, wenn ein PR-Report allein nicht reicht und dein Team eine gemeinsame Übersicht braucht.

## Bevor du startest

Das Dashboard brauchst du nur, wenn du eine gemeinsame Web-Ansicht möchtest.

- Wenn euch AI-Sitzungsberichte direkt in Pull Requests reichen, bleib bei [GitHub Action](./github-action/).
- Wenn ihr offene und gemergte Historie gemeinsam sehen wollt, geht es hier weiter.

## Schnellstart

1. Run Dashboard setup once.

```bash
npx agent-note init --agent claude --dashboard
```

2. Commit both workflow files.

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. In GitHub Pages, choose **GitHub Actions** as the source.

The generated Dashboard workflow restores data from `gh-pages`, refreshes it from `refs/notes/agentnote`, and publishes the shared `/dashboard/` view.

## Öffentliche URL

Typischerweise wird das Dashboard hier veröffentlicht:

- `https://<owner>.github.io/<repo>/dashboard/`

Zum Beispiel:

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Wenn du mit GitHub Pages veröffentlichst, übergibt dein Pages-Workflow die repo-spezifischen Werte für `SITE` und `BASE` zur Build-Zeit. In den meisten Repositories reicht das Beispiel-Workflow ohne weitere Änderungen am Dashboard source.
</Aside>

## Erstes Deploy

Auch ein neues Repository kann schon vor dem ersten Production-Deploy Dashboard-Daten sammeln. Die öffentliche Pages-URL erscheint aber erst nach dem ersten Deploy auf `default branch`.

## Wie es funktioniert

- `refs/notes/agentnote` remains the source of truth
- The generated Dashboard workflow writes shared snapshots to `gh-pages/dashboard/notes/*.json`
- GitHub Pages serves the built Dashboard at `/dashboard/`

The repository does not commit sample Dashboard JSON. A fresh Dashboard stays empty until the workflow restores or generates note files. `pull_request` refreshes open history, and `default branch` publishes the merged state.
