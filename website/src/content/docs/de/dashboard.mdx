## Was du bekommst

Das Dashboard gibt deinem Team eine gemeinsame URL für den Verlauf von Agent Note.

- Offene und gemergte PRs an einem Ort
- Commits, Prompts und Attribution ohne jede PR einzeln zu öffnen
- Eine gemeinsame GitHub Pages URL, normalerweise unter `/dashboard/`

Nutze es, wenn ein PR-Report allein nicht reicht und dein Team eine gemeinsame Übersicht braucht.

## Bevor du startest

Das Dashboard brauchst du nur, wenn du eine gemeinsame Web-Ansicht möchtest.

- Wenn euch AI-Sitzungsberichte direkt in Pull Requests reichen, bleib bei [GitHub Action](./github-action/).
- Wenn ihr offene und gemergte Historie gemeinsam sehen wollt, geht es hier weiter.

## Schnellstart

Lass zuerst die Action für jede PR Dashboard-Dateien schreiben:

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Dadurch werden für die aktuelle PR `packages/dashboard/public/notes/*.json` erstellt oder aktualisiert. Das Wiederherstellen, Speichern und Veröffentlichen übernimmt anschließend dein Pages-Workflow.

## Öffentliche URL

Typischerweise wird das Dashboard hier veröffentlicht:

- `https://<owner>.github.io/<repo>/dashboard/`

Zum Beispiel:

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Wenn du mit GitHub Pages veröffentlichst, übergibt dein Pages-Workflow die repo-spezifischen Werte für `SITE` und `BASE` zur Build-Zeit. In den meisten Repositories reicht das Beispiel-Workflow ohne weitere Änderungen an `packages/dashboard`.
</Aside>

## Erstes Deploy

Auch ein neues Repository kann schon vor dem ersten Production-Deploy Dashboard-Daten sammeln. Die öffentliche Pages-URL erscheint aber erst nach dem ersten Deploy auf `main`.

## Wie es funktioniert

- `packages/dashboard/` ist die Astro-App für die Ansicht
- `packages/dashboard/public/notes/*.json` ist das Bundle, das beim Build gelesen wird
- `gh-pages/dashboard/notes/*.json` ist der dauerhafte Speicher für den gemeinsamen Verlauf

Das Repository enthält keine Beispiel-JSON für das Dashboard. Ein neues Dashboard bleibt leer, bis dein Workflow Note-Dateien wiederherstellt oder erzeugt. In vielen Setups werden PR-Notes bei `pull_request` aktualisiert und der gemergte Stand bei `main` veröffentlicht.
