## Schnellstart

Nutze einfach den Workflow, den `agent-note init` anlegt:

```yaml
name: Agent Note
on:
  pull_request:
    types: [opened, synchronize]
permissions:
  contents: read
  pull-requests: write
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: wasabeef/AgentNote@v0
```

Das reicht. Jede PR bekommt automatisch einen KI-Sitzungsbericht.

## Eingaben

| Input | Standard | Beschreibung |
|---|---|---|
| `base` | Basis-Branch der PR | Branch, gegen den verglichen wird |
| `pr_output` | `description` | Ziel des PR-Berichts: `description`, `comment` oder `none` |
| `dashboard` | `false` | Schreibt zusätzlich Dashboard-Dateien für die aktuelle PR |

### PR-Beschreibung und Dashboard zusammen nutzen

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

<Aside type="tip">
  Aktiviere `dashboard: true` nur, wenn du auch das gemeinsame Dashboard nutzen willst. Wie Bundle-Dateien, Pages-Sync und Deploy zusammenspielen, erklärt die Seite <a href="./dashboard/">Dashboard</a>.
</Aside>

## Ausgaben

| Output | Typ | Beschreibung |
|---|---|---|
| `overall_ai_ratio` | number | KI-Anteil über die gesamte PR (0-100) |
| `overall_method` | string | `line`, `file`, `mixed` oder `none` |
| `tracked_commits` | number | Anzahl der Commits mit agent-note Daten |
| `total_commits` | number | Gesamtzahl der Commits in der PR |
| `total_prompts` | number | Gesamtzahl der Prompts über alle Commits |
| `json` | string | Vollständiger strukturierter Bericht |
| `markdown` | string | Gerenderter Markdown-Bericht |
| `dashboard_commits` | number | Anzahl der ins Dashboard-Bundle geschriebenen tracked Commits |

## Bericht

Der PR-Bericht enthält:

- **Header** — `Total AI Ratio` mit Balken sowie darunter `Model`
- **Tabelle** — KI-Anteil, Prompt-Anzahl und Dateien pro Commit
- **Prompts** — einklappbare Prompt/Response-Paare pro Commit

<Aside type="tip">
  Commit-Hashes im Bericht verlinken direkt auf die Commit-Seite bei GitHub.
</Aside>
