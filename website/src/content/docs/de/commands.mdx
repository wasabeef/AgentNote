Diese Seite zeigt die Befehle, die Nutzer direkt ausführen. Für das erste Setup starte mit [Installation](./installation/).

## Häufige Aufgaben

- Erstes Setup: `agent-note init`
- Agent Note aus dem Repository entfernen: `agent-note deinit`
- Status prüfen: `agent-note status`
- Neue Aktivität prüfen: `agent-note log`
- Commit-Daten anzeigen: `agent-note show`
- Verstehen, warum eine Zeile geändert wurde: `agent-note why`

<Aside>
  PR Report und Dashboard sind GitHub-Action-Funktionen. Sie werden normalerweise durch `agent-note init` erzeugt, nicht durch interne Report-Befehle.
</Aside>

## agent-note init

Richtet Agent Note im Repository ein.

```bash
npx agent-note init --agent claude
```

| Flag | Description |
|---|---|
| `--agent <name...>` | Choose one or more agent adapters: `claude`, `codex`, `cursor`, or `gemini` |
| `--dashboard` | Also create the shared Dashboard workflow |
| `--no-hooks` | Skip agent hook configuration |
| `--no-git-hooks` | Skip git hook installation |
| `--no-action` | Skip workflow creation |
| `--no-notes` | Skip notes auto-fetch config |
| `--hooks` | Only install agent hooks |
| `--action` | Only create workflow |

## agent-note deinit

Entfernt Agent-Note-Hooks und Konfiguration für einen oder mehrere Agents.

```bash
npx agent-note deinit --agent claude
```

| Flag | Beschreibung |
|---|---|
| `--agent <name...>` | `claude`, `codex`, `cursor`, or `gemini` |
| `--remove-workflow` | Entfernt auch die PR-Report- und Dashboard-Workflows |
| `--keep-notes` | Behält die notes-auto-fetch-Konfiguration |

## agent-note status

Zeigt den Tracking state.

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

## agent-note log

Listet aktuelle Commits mit AI Ratio.

```bash
npx agent-note log
npx agent-note log 20
```

```text
ce941f7 feat: add auth  [a1b2c3d4… | 🤖60% | 2p]
326a568 test: add tests [a1b2c3d4… | 🤖100% | 1p]
ba091be fix: update deps
```

## agent-note show

Zeigt die KI-Session eines Commits.

```bash
npx agent-note show
npx agent-note show abc1234
```

```text
commit:  ce941f7 feat: add JWT auth middleware
session: a1b2c3d4-5678-90ab-cdef-111122223333

ai:      60% (45/75 lines) [█████░░░]
model:   claude-sonnet-4-20250514
agent:   claude
files:   5 changed, 3 by AI

  src/middleware/auth.ts  🤖
  src/types/token.ts  🤖
  CHANGELOG.md  👤

prompts: 2

  1. Implement JWT auth middleware with refresh token rotation
  2. Add tests for expired token and invalid signature
```

## agent-note why

Zeigt den Agent Note context hinter einer geänderten Zeile.

```bash
npx agent-note why src/middleware/auth.ts:42
npx agent-note why src/middleware/auth.ts:42-55
npx agent-note why src/middleware/auth.ts:42:7
npx agent-note why src/middleware/auth.ts#L42
npx agent-note why src/middleware/auth.ts#L42-L55
npx agent-note why src/middleware/auth.ts#L42C7
npx agent-note why @src/middleware/auth.ts#L42
npx agent-note why https://github.com/owner/repo/blob/main/src/middleware/auth.ts#L42
npx agent-note why file:///workspace/repo/src/middleware/auth.ts#L42
npx agent-note why vscode://file/workspace/repo/src/middleware/auth.ts:42:7
```

<Aside>
  `why` akzeptiert `path:line`, `path#Lline`, GitHub file URLs, `file://...`, `vscode://file...` und paths mit führendem `@` aus AI-Agent-Ausgaben.
</Aside>

## AI Agent Skill

Installiere den Agent Note Skill, um diese Aufgaben in natürlicher Sprache anzustoßen.

```bash
gh skill install wasabeef/AgentNote agent-note --agent codex --scope user
```

Wähle den passenden `--agent` Wert: `codex`, `claude-code`, `cursor` oder `gemini-cli`.
