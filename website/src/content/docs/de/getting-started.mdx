Agent Note lässt sich leichter einführen, wenn die Bausteine vorher klar sind. Agent-Hooks erfassen den lokalen Kontext, git notes hängen ihn an Commits, und die GitHub Action macht daraus einen PR Report.

Diese Seite ist die Karte. Die Installationsbefehle stehen auf der nächsten Seite, damit du zuerst weißt, was du aktivierst.

## Was du einrichtest

<CardGrid>
  <Card title="Lokale Erfassung" icon="laptop">
    Agent-Hooks sammeln Prompts, Antworten, berührte Dateien und Attribution-Signale während der Arbeit.
  </Card>
  <Card title="Git-native Historie" icon="document">
    Jeder Commit erhält eine git note unter `refs/notes/agentnote`, ohne den Branch-Inhalt zu verändern.
  </Card>
  <Card title="Pull-Request-Kontext" icon="rocket">
    Die GitHub Action liest diese Notes und schreibt den PR Report für Reviewer.
  </Card>
  <Card title="Optionales Dashboard" icon="setting">
    Wenn das Team eine gemeinsame Ansicht möchte, veröffentlicht Dashboard die Historie auf GitHub Pages.
  </Card>
</CardGrid>

## Empfohlener Weg

<Steps>
1. Lies zuerst dieses kurze Modell.
2. Folge [Installation](./installation/), um `npx agent-note init` auszuführen.
3. Öffne eine Pull Request und prüfe den PR Report.
4. Aktiviere Dashboard nur, wenn dein Team eine gemeinsame Seite braucht.
</Steps>
