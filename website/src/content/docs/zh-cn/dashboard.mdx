## 你会得到什么

Dashboard 会给团队一个共享 URL，用来查看 Agent Note 的历史记录。

- 在一个地方查看 open 和 merged 的 PR 历史
- 不用逐个打开 PR，也能查看 commit、prompt 和 attribution
- 一个共享的 GitHub Pages URL，通常位于 `/dashboard/`

当 PR report 已经不够用，而团队需要一个统一的查看页面时，就适合使用 Dashboard。

## 开始前

只有在你需要共享的网页视图时，才需要 Dashboard。

- 如果 pull request 里的 AI 会话报告已经足够，就停在 [GitHub Action](./github-action/)。
- 如果你想要一个共享的 Pages 视图来查看 open 和 merged 历史，就继续看这里。

## 快速设置

首先，让 Action 为每个 PR 写出 Dashboard 文件：

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

这样会为当前 PR 创建或更新 `packages/dashboard/public/notes/*.json`。之后由你的 Pages workflow 负责恢复、保存和发布这些文件。

## 公共 URL

通常 Dashboard 会发布在这里：

- `https://<owner>.github.io/<repo>/dashboard/`

例如：

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  如果你通过 GitHub Pages 发布，Pages workflow 会在 build 时传入仓库专用的 `SITE` 和 `BASE`。对大多数仓库来说，直接使用示例 workflow 就足够，不需要额外修改 `packages/dashboard`。
</Aside>

## 首次部署

即使是全新的仓库，也可以在第一次 production deploy 之前先积累 Dashboard 数据。公开的 Pages URL 则要等第一次部署到 `main` 之后才会出现。

## 工作方式

- `packages/dashboard/` 是用于展示 Dashboard 的 Astro 应用
- `packages/dashboard/public/notes/*.json` 是 build 时读取的 bundle
- `gh-pages/dashboard/notes/*.json` 是共享历史的持久存储

仓库本身不包含 Dashboard 示例 JSON。新的 Dashboard 会保持为空，直到 workflow 恢复或生成 note files。很多团队会在 `pull_request` 时更新 PR notes，再从 `main` 发布 merged 状态。
