## 快速设置

在你的仓库里先执行一次：

```bash
npx agent-note init --agent claude
```

这样会为你创建 PR 报告 workflow：

```yaml
# .github/workflows/agentnote-pr-report.yml
name: Agent Note PR Report
on:
  pull_request:
    types: [opened, reopened, synchronize]
permissions:
  contents: read
  pull-requests: write
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0
      - uses: wasabeef/AgentNote@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

这样就完成了。每个 PR 都会得到一份 AI 会话报告。

## Inputs

| Input | Default | 说明 |
|---|---|---|
| `base` | PR 的 base branch | 用于比较的分支 |
| `pr_output` | `description` | PR 报告的输出位置：`description`、`comment` 或 `none` |
| `prompt_detail` | `standard` | PR 报告中 prompt 历史的详细程度：`compact` = 仅 high，`standard` = high + medium，`full` = 所有已保存 prompt |
| `dashboard` | `false` | 运行 Dashboard Mode，而不是 PR Report Mode。通常由 `agent-note init --dashboard` 生成 |

## Dashboard

```bash
npx agent-note init --agent claude --dashboard
```

<Aside type="tip">
  这也会额外添加 `.github/workflows/agentnote-dashboard.yml`，用来配置共享 Dashboard。
</Aside>

## Outputs

| Output | 类型 | 说明 |
|---|---|---|
| `overall_ai_ratio` | number | 整个 PR 的 AI ratio (0-100) |
| `overall_method` | string | `line`、`file`、`mixed` 或 `none` |
| `tracked_commits` | number | 带有 agent-note 数据的 commit 数 |
| `total_commits` | number | PR 中的 commit 总数 |
| `total_prompts` | number | 所有 commit 的 prompt 总数 |
| `json` | string | 完整的结构化报告 |
| `markdown` | string | 已渲染的 Markdown 报告 |
| `should_deploy` | boolean string | Dashboard Mode 输出，用来告诉调用方 workflow 是否需要发布 Pages |

## 报告内容

PR 报告包含：

- **Header** — `Total AI Ratio` 与进度条，以及下一行的 `Model`
- **表格** — 每个 commit 的 AI ratio、prompt 数和文件列表
- **Prompts** — 可折叠的 `📝 Context`、`🧑 Prompt`、`🤖 Response` 区块

<Aside type="tip">
  报告里的 commit hash 会直接链接到 GitHub 上对应的 commit 页面。
</Aside>
