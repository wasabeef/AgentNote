## 快速设置

Run this once in your repository:

```bash
npx agent-note init --agent claude
```

That creates the PR report workflow for you:

```yaml
# .github/workflows/agentnote-pr-report.yml
name: Agent Note PR Report
on:
  pull_request:
    types: [opened, reopened, synchronize]
permissions:
  contents: read
  pull-requests: write
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v6
        with:
          node-version: 22
          cache: npm
      - uses: wasabeef/AgentNote@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

That's it. Every PR gets an AI session report.

## Inputs

| Input | Default | 说明 |
|---|---|---|
| `base` | PR 的 base branch | 用于比较的分支 |
| `pr_output` | `description` | PR 报告的输出位置：`description`、`comment` 或 `none` |

## Dashboard

```bash
npx agent-note init --agent claude --dashboard
```

<Aside type="tip">
  This also adds `.github/workflows/agentnote-dashboard.yml` for the shared Dashboard setup.
</Aside>

## Outputs

| Output | 类型 | 说明 |
|---|---|---|
| `overall_ai_ratio` | number | 整个 PR 的 AI ratio (0-100) |
| `overall_method` | string | `line`、`file`、`mixed` 或 `none` |
| `tracked_commits` | number | 带有 agent-note 数据的 commit 数 |
| `total_commits` | number | PR 中的 commit 总数 |
| `total_prompts` | number | 所有 commit 的 prompt 总数 |
| `json` | string | 完整的结构化报告 |
| `markdown` | string | 已渲染的 Markdown 报告 |

## 报告内容

PR 报告包含：

- **Header** — `Total AI Ratio` 与进度条，以及下一行的 `Model`
- **表格** — 每个 commit 的 AI ratio、prompt 数和文件列表
- **Prompts** — 可折叠的 prompt / response 区块

<Aside type="tip">
  报告里的 commit hash 会直接链接到 GitHub 上对应的 commit 页面。
</Aside>
