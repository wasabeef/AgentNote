## Agent Note 能做什么

<CardGrid stagger>
  <Card title="行级 AI 贡献度分析" icon="pencil">
    Claude Code 提供完整的行级归因。Codex 会在 transcript 中的 patch 行数与最终 commit 一致时提升到行级。
  </Card>
  <Card title="会话追踪" icon="document">
    提示词、响应、模型名称和工具使用记录——全部关联到对应的提交。
  </Card>
  <Card title="PR 报告" icon="rocket">
    在 Pull Request 上自动生成 AI 会话报告，包含每次提交的比例、行数和关联哈希。
  </Card>
  <Card title="Git 原生" icon="setting">
    以 git notes 形式存储——对分支和 CI 不可见。可推送、可拉取、可共享。
  </Card>
</CardGrid>

## 60 秒完成配置

<Steps>
1. 在仓库中运行 init

   ```bash
   npx agentnote init --agent claude
   ```

   如果你使用 Codex CLI：`npx agentnote init --agent codex`
   如果你使用 Cursor：`npx agentnote init --agent cursor`
   Gemini CLI 请运行：`npx agentnote init --agent gemini`
   同时使用多个 agent：`npx agentnote init --agent claude --agent cursor`

2. 提交生成的文件

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agentnote"
   git push
   ```

   Codex 仓库需要提交 `.codex/config.toml` 和 `.codex/hooks.json`，而不是 `.claude/settings.json`。
   Cursor 仓库需要提交 `.cursor/hooks.json`。
   Gemini CLI 仓库提交 `.gemini/settings.json`。

3. Claude Code 和 Codex 可以照常使用 `git commit`。
   如果已安装生成的 git hook，Cursor preview 也可以直接使用普通 `git commit`。
   如果 git hook 不可用，请使用 `agentnote commit -m "..."` 作为 fallback。
</Steps>

## 支持情况

| Agent | 状态 | 说明 |
|---|---|---|
| Claude Code | 完整支持 | 通过 hook 捕获的 blob 对实现行级归因 |
| Codex CLI | Preview | 基于 transcript；当 patch 行数与 commit 一致时提升到行级 |
| Cursor | Preview | 基于 `afterFileEdit` / `afterTabFileEdit`；仅在最终 blob 仍与 AI edit 一致时安全升级到行级归因 |
| Gemini CLI | Preview | 通过 `.gemini/settings.json` 中的 `BeforeTool`/`AfterTool` 钩子进行文件级归属判定 |
