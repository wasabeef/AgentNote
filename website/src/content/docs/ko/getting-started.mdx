Agent Note 는 먼저 구성 요소를 이해하면 훨씬 쉽게 도입할 수 있습니다. local hook 이 context 를 모으고, git notes 가 commit 에 붙이며, GitHub Action 이 이를 PR Report 로 만듭니다.

이 페이지는 전체 지도입니다. 설치 명령은 다음 페이지에 두어, 무엇을 활성화하는지 먼저 확인할 수 있게 했습니다.

## 설정되는 것

<CardGrid>
  <Card title="Local capture" icon="laptop">
    agent hook 이 작업 중 prompt, response, touched file, attribution signal 을 수집합니다.
  </Card>
  <Card title="Git-native history" icon="document">
    각 commit 에 `refs/notes/agentnote` 의 git note 를 붙이며 branch 내용은 바꾸지 않습니다.
  </Card>
  <Card title="Pull Request context" icon="rocket">
    GitHub Action 이 notes 를 읽고 reviewer 를 위한 PR Report 를 작성합니다.
  </Card>
  <Card title="Optional Dashboard" icon="setting">
    팀이 공유 화면을 원하면 Dashboard 를 GitHub Pages 에 게시할 수 있습니다.
  </Card>
</CardGrid>

<figure class="doc-screenshot">
  <img src="/AgentNote/images/dashboard-preview.png" alt="Agent Note Dashboard showing Pull Request history, commits, prompts, and diffs" />
  <figcaption>Dashboard 는 optional 이지만, 일반 commit 과 Pull Request 에서 Agent Note 가 어떤 review history 를 만들 수 있는지 보여줍니다.</figcaption>
</figure>

## 추천 흐름

<Steps>
1. 이 짧은 모델을 먼저 읽습니다.
2. [설치](./installation/)에서 `npx agent-note init` 을 실행합니다.
3. Pull Request 를 열고 PR Report 를 확인합니다.
4. 팀 공유 화면이 필요할 때만 Dashboard 를 추가합니다.
</Steps>
