## 무엇을 할 수 있나

Dashboard 는 팀이 Agent Note 기록을 함께 볼 수 있는 하나의 공유 URL 입니다.

- open / merged 된 PR 기록을 한곳에서 볼 수 있습니다
- 각 PR 을 열지 않고도 commit, prompt, attribution 을 확인할 수 있습니다
- 보통 `/dashboard/` 아래의 GitHub Pages URL 하나를 팀과 공유할 수 있습니다

PR Description 만으로는 부족하고, 팀 전체가 보는 목록 화면이 필요할 때 사용하면 좋습니다.

## 시작 전에

공유된 웹 화면이 필요할 때만 Dashboard 를 쓰면 됩니다.

- pull request 안의 AI session report 만으로 충분하다면 [GitHub Action](./github-action/) 에서 멈추면 됩니다.
- open / merged 기록을 함께 보는 Pages 화면이 필요하다면 여기서 계속 진행하면 됩니다.

## Quick Setup

먼저 Action 이 PR 별 Dashboard bundle 을 쓰도록 설정합니다.

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

이렇게 하면 현재 PR 에 대해 `packages/dashboard/public/notes/*.json` 이 생성 또는 갱신됩니다. 이후 Pages workflow 가 이 파일을 restore, persist, deploy 합니다.

## 공개 URL

보통 dashboard 는 다음 URL 에 공개합니다.

- `https://<owner>.github.io/<repo>/dashboard/`

예시:

- `https://wasabeef.github.io/AgentNote/dashboard/`

## Deploy 설정

`packages/dashboard` 는 로컬 default 로 `/` 를 유지하므로, Pages workflow 쪽에서 build 시 repo 전용 값을 주입해야 합니다.

- `SITE=https://<owner>.github.io`
- `BASE=/<repo>/dashboard`
- `PUBLIC_REPO=<owner>/<repo>`
- `PUBLIC_REPO_URL=https://github.com/<owner>/<repo>` (optional)

이렇게 하면 package 자체는 generic 하게 유지하면서도, repo 별 public URL 을 정확하게 만들 수 있습니다.

## 권장 흐름

1. GitHub Action 에서 `dashboard: true` 를 켭니다.
2. `gh-pages/dashboard/notes` 를 `packages/dashboard/public/notes` 로 restore 합니다.
3. `pull_request` (`opened`, `reopened`, `synchronize`) 에서 해당 PR 의 note files 를 다시 쓰고 `gh-pages` 로 persist 합니다.
4. `main` 으로 `push` 되면 해당 PR notes 를 `merged` 로 갱신하고, site 를 다시 build / deploy 합니다.

이 흐름이면 `main` 을 generated JSON 으로 더럽히지 않으면서도, 하나의 shared Dashboard 에서 open / merged PR 기록을 모두 볼 수 있습니다.

## 첫 deploy

새 repo 도 첫 production deploy 전까지 Dashboard data 를 쌓을 수 있지만, public Pages URL 은 첫 `main` deploy 이후에 생깁니다.

## 동작 방식

- `packages/dashboard/` 는 interactive Astro app 입니다
- `packages/dashboard/public/notes/*.json` 는 build 때 읽는 working bundle 입니다
- `gh-pages/dashboard/notes/*.json` 는 shared 기록을 위한 durable store 입니다

이 repository 는 sample Dashboard JSON 을 포함하지 않습니다. 새 Dashboard 는 workflow 가 note files 를 restore 하거나 생성하기 전까지 비어 있습니다.
