### 세션 파일

| 파일 | 목적 |
|---|---|
| `prompts.jsonl` | turn 번호와 함께 한 줄에 하나의 prompt |
| `changes.jsonl` | AI 가 수정한 파일과 post-edit blob hash |
| `pre_blobs.jsonl` | 각 AI edit 이전의 file 상태 |
| `heartbeat` | 세션 활동 시각 |
| `turn` | prompt 마다 증가하는 단조 증가 카운터 |

## `📝 Context` 를 수집하는 방식

`📝 Context` 는 `continue` 나 `다음 작업으로 넘어가` 같은 짧은 prompt 를 읽기 쉽게 만드는 display-only metadata 입니다. attribution 에는 영향을 주지 않습니다.

<img src="/AgentNote/images/context-dashboard-example.png" alt="Agent Note Dashboard showing Context before a short prompt" style="width: 100%; border-radius: 20px; border: 1px solid rgba(148, 163, 184, 0.18); box-shadow: 0 24px 80px rgba(15, 23, 42, 0.28);" />

- `reference` context 는 바로 이전 response 에서 가져옵니다. final diff 의 changed file path, file basename, code identifier 같은 강한 structural anchor 가 있을 때만 사용합니다.
- `scope` context 는 현재 response 에서 가져옵니다. response 초반이 structural anchor 로 작업 범위를 설명할 때만 사용합니다.
- Agent Note 는 `yes`, `continue`, `お願いします` 같은 approval keyword 를 선택 규칙으로 사용하지 않습니다. structural anchor 가 없으면 Context 를 붙이지 않습니다.
- Context 는 `files_touched`, prompt ownership, AI ratio, attribution method, git note 저장 여부를 절대 바꾸지 않습니다.

## Agent Note vs Entire

Agent Note 와 Entire 는 비슷한 문제를 다루지만 목표 workflow 가 다릅니다.

- Agent Note 는 git commit 과 그 commit 을 만든 AI conversation 을 연결합니다. Entire 는 rewind / resume workflow 를 포함하는 더 넓은 checkpoint product 입니다.
- Agent Note 의 영구 record 는 `refs/notes/agentnote` 에 저장되고, 임시 session file 은 `.git/agentnote/` 아래에 남습니다. Entire 는 checkpoint metadata 와 hosted web application 을 중심으로 합니다.
- Agent Note Dashboard 는 git notes 기반의 optional static GitHub Pages 입니다. hosted service 없이도 Agent Note 를 사용할 수 있습니다.
- Agent Note 는 과거 workspace state 를 복원하려는 tool 이 아닙니다. commit 이 왜 바뀌었는지, 어떤 prompt 가 관련됐는지, diff 중 어느 정도가 AI-authored 인지 review 하기 쉽게 만드는 데 집중합니다.

<Aside type="tip">
  prompt 목록은 session 전체가 아니라 commit 과 다음 commit 사이의 대화 window 에서 가져옵니다. Agent Note 는 현재 commit 을 설명하는 planning, clarification, review prompt 를 남기고, 인용된 과거 prompt 기록이나 이 commit 에 남지 않은 edit turn 은 잘라냅니다.
</Aside>

<Aside>
  대표적인 generated artifact 는 framework/cache path, generated filename/suffix, commit 된 file header 같은 구체적인 signal 로 best-effort 로 `AI ratio` 분모에서 제외됩니다. `build/`, `dist/`, `gen/`, `generated/`, `target/` 같은 범용 directory 이름만으로는 generated 로 보지 않습니다.
</Aside>

## 세션 라이프사이클

| 이벤트 | 발생하는 일 |
|---|---|
| **SessionStart** | 세션 디렉터리를 만들고 heartbeat 를 기록 |
| **UserPromptSubmit** | prompt 를 추가하고, turn counter 를 증가시키며, log 를 rotate |
| **PreToolUse Edit/Write** | pre-edit blob hash 를 동기적으로 캡처 |
| **PostToolUse Edit/Write** | post-edit blob hash 와 file path 를 캡처 |
| **Stop** | stop event 를 기록함 (heartbeat 는 계속 활성 상태 — Stop = response 끝, session 끝 아님) |

<Aside>
  git hook 은 trailer 를 주입하거나 notes 를 기록하기 전에 heartbeat 로 세션 freshness 를 확인합니다. heartbeat 는 SessionStart 와 각 prompt 에서 업데이트됩니다.
</Aside>

<Aside type="tip">
  Agent Note 는 현재 agent 를 인지합니다. 세션 메타데이터에 active adapter 가 저장되므로, 나중에 `show`, `status`, commit recording 이 올바른 transcript format 을 해석할 수 있습니다.
</Aside>

## Git Notes

Git notes 는 branch 목록, GitHub UI, CI 에서는 보이지 않지만 push / fetch 는 가능합니다.

```bash
# 팀과 공유
git push origin refs/notes/agentnote

# remote 에서 가져오기
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# note 보기
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` 이 auto-fetch 를 설정하고, `pre-push` hook 이 auto-push 를 처리합니다. 수동 명령은 필요하지 않습니다.
</Aside>
