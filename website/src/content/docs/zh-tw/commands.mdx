當你已經理解 Agent Note 的流程，只需要查找具體 command 或 flag 時，可以使用這一頁。首次 setup 請從 [安裝](./installation/) 開始。

## 常用任務

- 首次 setup: `agent-note init`
- 查看 commit 記錄了什麼: `agent-note show`
- 查看某一行為什麼改變: `agent-note why`
- 查看最近活動: `agent-note log`
- 產生 PR Report: `agent-note pr`
- 檢查 hooks 和 active adapter: `agent-note status`

## agent-note init

```bash
npx agent-note init --agent claude
```

| Flag | 說明 |
|---|---|
| `--agent <name...>` | 必填。一個或多個 adapter: `claude`, `codex`, `cursor`, `gemini` |
| `--dashboard` | 同時建立 shared Dashboard workflow |
| `--no-hooks` | 跳過 agent hook 設定 |
| `--no-git-hooks` | 跳過 git hook 安裝 |
| `--no-action` | 跳過 workflow 建立 |
| `--no-notes` | 跳過 notes auto-fetch 設定 |
| `--hooks` | 只安裝 agent hooks |
| `--action` | 只建立 workflow |

## agent-note show

```bash
npx agent-note show
npx agent-note show abc1234
```

顯示某個 commit 的 prompts、responses、touched files、agent、model 和 attribution。

## agent-note why

使用 `git blame` 找到最後修改某一行的 Commit，並顯示該 Commit 對應的 Agent Note context。

```bash
npx agent-note why src/middleware/auth.ts:42
npx agent-note why src/middleware/auth.ts:42-55
```

<Aside>
  `why` 縮短了手動執行 `git blame` → `git notes --ref=agentnote show <commit>` 的流程。它顯示 Commit 或 file 層級的 prompt evidence，但不會宣稱知道寫下該行的精確 prompt。
</Aside>

## agent-note log

```bash
npx agent-note log
npx agent-note log 20
```

列出最近 commit，並顯示 AI ratio 和保存的 prompt 數量。

## agent-note pr

<Tabs>
  <TabItem label="Markdown">
  ```bash
  npx agent-note pr origin/main
  ```
  </TabItem>
  <TabItem label="Detail">
  ```bash
  npx agent-note pr origin/main --prompt-detail compact
  ```
  </TabItem>
  <TabItem label="JSON">
  ```bash
  npx agent-note pr origin/main --json
  ```
  </TabItem>
</Tabs>

<Aside>
  `compact` 會讓 PR Report 聚焦在解釋 commit 的 prompts。`full` 會顯示所有保存的 prompts。
</Aside>

## agent-note status

```bash
npx agent-note status
```

檢查 active adapter、hooks 收集的內容、git hooks 狀態和 commit 記錄路徑。

## agent-note deinit

```bash
npx agent-note deinit --agent claude
```

若也要刪除 PR Report 和 Dashboard workflow，請加上 `--remove-workflow`。

## agent-note commit

```bash
npx agent-note commit -m "feat: add auth"
```

<Aside type="tip">
  如果 git hooks 已安裝，則不需要它。一般 `git commit` 已經會記錄同樣的資料。
</Aside>
