## 它能做什麼

<CardGrid stagger>
  <Card title="保留 AI 編寫行的紀錄" icon="pencil">
    比 `git blame` 更細粒度。Claude Code 完全支援行級；其他 agent 在條件符合時升級到行級。
  </Card>
  <Card title="提示詞和回應一併儲存" icon="document">
    模型名稱和工具使用情況也一起——全部與變更捆綁存入 git notes。
  </Card>
  <Card title="自動產生 PR 報告" icon="rocket">
    AI 比例、提示詞數量、commit 雜湊值。審查者在看程式碼之前就能看到上下文。
  </Card>
  <Card title="僅寫入 git notes" icon="setting">
    分支保持乾淨。push / fetch 與團隊共享，也可以在 GitHub Pages 上發佈可選的靜態 dashboard。
  </Card>
</CardGrid>

## 設定

<Steps>
1. 在儲存庫中執行 init

   ```bash
   npx agent-note init --agent claude
   ```

   - 如果你使用 Codex CLI：`npx agent-note init --agent codex`
   - 如果你使用 Cursor：`npx agent-note init --agent cursor`
   - Gemini CLI 請執行：`npx agent-note init --agent gemini`
   - 同時使用多個 agent：`npx agent-note init --agent claude --agent cursor`

2. 提交產生的檔案

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   - Codex CLI: 提交 `.codex/config.toml` 與 `.codex/hooks.json`
   - Cursor: 提交 `.cursor/hooks.json`
   - Gemini CLI: 提交 `.gemini/settings.json`

3. 用 AI 編寫程式碼。
   安裝產生的 git hook 後，普通 `git commit` 即可正常運作。
   若 git hook 無法使用，`agent-note commit -m "..."` 仍可作為 fallback。
</Steps>

## 檢查設定

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` 顯示已啟用的 adapter，`capture:` 彙總活躍 agent hook 收集的內容，`git:` 顯示 repo 本地 git hook 是否已安裝，`commit:` 顯示普通 `git commit` 是否為主路徑或 fallback 模式是否啟用。
