## 資料流程

  <div class="agent-flow__step">

  <div class="agent-flow__arrow" aria-hidden="true">→</div>

  </div>
  <div class="agent-flow__arrow" aria-hidden="true">→</div>
  <div class="agent-flow__step">
    <span class="agent-flow__label">4. Share</span>
    <strong>隨程式碼共享</strong>

只要生成的 git hook 已安裝，大多數儲存庫裡就繼續正常使用 `git commit` 與 `git push` 即可。Cursor Preview 也能從本機 transcript 還原 prompt / response，在 git hook 無法使用的環境下也能繼續追蹤 commit。

### 工作階段檔案

| 檔案 | 用途 |
|---|---|
| `prompts.jsonl` | 每行一個 prompt，並附帶 turn 編號 |
| `changes.jsonl` | AI 編輯過的檔案，以及 post-edit blob hashes |
| `pre_blobs.jsonl` | 每次 AI edit 前的檔案狀態 |
| `heartbeat` | 工作階段活動時間戳 |
| `turn` | 每次 prompt 都會遞增的單調計數器 |

### Note Schema

```json
{
  "v": 1,
  "agent": "claude",
  "session_id": "a1b2c3d4-...",
  "model": "claude-sonnet-4-20250514",
  "interactions": [
    {
      "prompt": "實作 JWT auth middleware",
      "contexts": [
        { "kind": "reference", "source": "previous_response", "text": "上一則 response 說明了為什麼這個 middleware 需要修改。" }
      ],
      "response": "我會建立這個 middleware...",
      "files_touched": ["src/auth.ts"],
      "tools": ["Edit"]
    }
  ],
  "files": [
    { "path": "src/auth.ts", "by_ai": true },
    { "path": "CHANGELOG.md", "by_ai": false }
  ],
  "attribution": {
    "ai_ratio": 73,
    "method": "line",
    "lines": { "ai_added": 146, "total_added": 200, "deleted": 12 }
  }
}
```

<Aside type="tip">
  prompt 清單來自 commit 到下一個 commit 之間的對話 window，而不是整個 session backlog。Agent Note 會保留能解釋目前 commit 的 planning、clarification 與 review prompt，同時裁掉被引用的舊 prompt 歷史，以及沒有保留到這個 commit 中的 edit turn。如果很短的 prompt 依賴上一則 response，但上一個 turn 已經屬於更早的 commit，Agent Note 可能會在 prompt 前加入只用於顯示的 `context` 欄位；它不會改變 attribution。
</Aside>

<Aside>
  常見的 generated artifacts 會以 best-effort 的方式根據具體 signal 排除在 `AI ratio` 的分母之外，例如 framework/cache path、generated filename/suffix，以及 committed file header。僅有 `build/`、`dist/`、`gen/`、`generated/`、`target/` 這類通用目錄名並不足以判定為 generated。
</Aside>

## Session 生命週期

| 事件 | 發生內容 |
|---|---|
| **SessionStart** | 建立 session 目錄並寫入 heartbeat |
| **UserPromptSubmit** | 追加 prompt、遞增 turn counter，並輪換 logs |
| **PreToolUse Edit/Write** | 同步擷取 pre-edit blob hash |
| **PostToolUse Edit/Write** | 擷取 post-edit blob hash 與檔案路徑 |
| **Stop** | 記錄 stop 事件（heartbeat 會保持有效 —— Stop = response 結束，不是 session 結束） |

<Aside>
  Git hooks 在注入 trailers 或記錄 notes 之前，會透過 heartbeat 驗證 session 是否仍然新鮮。heartbeat 會在 SessionStart 與每次 prompt 時更新。
</Aside>

<Aside type="tip">
  Agent Note 會識別目前的 agent。session metadata 會保存目前的 adapter，因此 `show`、`status` 與 commit recording 之後都能解析正確的 transcript format。
</Aside>

## Git Notes

Git notes 不會出現在 branch 清單、GitHub UI 或 CI 中，但依然可以 push 與 fetch。

```bash
# 與團隊共享
git push origin refs/notes/agentnote

# 從 remote 取回
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# 查看 note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` 會設定 auto-fetch，而 `pre-push` hook 會處理 auto-push。不需要手動命令。
</Aside>
