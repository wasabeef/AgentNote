## 資料流程

<div class="agent-flow" role="img" aria-label="Agent Note 從 prompt 到共享 git notes 的流程">
  <div class="agent-flow__step">

只要 repo-local git hooks 已安裝，就不需要額外命令。Cursor preview 也能從 response hooks 或本機 transcripts 還原 prompt / response 配對；當 git hooks 無法使用時，它的 shell hooks 也會提供 fallback 路徑。

### 工作階段檔案

| 檔案 | 用途 |
|---|---|
| `prompts.jsonl` | 每行一個 prompt，並附帶 turn 編號 |
| `changes.jsonl` | AI 編輯過的檔案，以及 post-edit blob hashes |
| `pre_blobs.jsonl` | 每次 AI edit 前的檔案狀態 |
| `heartbeat` | 工作階段活動時間戳 |
| `turn` | 每次 prompt 都會遞增的單調計數器 |

### Note Schema

```json
{
  "v": 1,
  "agent": "claude",
  "session_id": "a1b2c3d4-...",
  "model": "claude-sonnet-4-20250514",
  "interactions": [
    {
      "prompt": "實作 JWT auth middleware",
      "response": "我會建立這個 middleware...",
      "files_touched": ["src/auth.ts"],
      "tools": ["Edit"]
    }
  ],
  "files": [
    { "path": "src/auth.ts", "by_ai": true },
    { "path": "CHANGELOG.md", "by_ai": false }
  ],
  "attribution": {
    "ai_ratio": 73,
    "method": "line",
    "lines": { "ai_added": 146, "total_added": 200, "deleted": 12 }
  }
}
```

<Aside type="tip">
  prompt 清單來自最終 diff 周圍具有因果關係的對話，而不是整個 session backlog。在 file-level 模式下，Agent Note 只保留最後一條 edit 路徑，以及通往它的 prompt-only context。
</Aside>

<Aside>
  常見的 generated artifacts 會以 best-effort 的方式排除在 `AI ratio` 的分母之外，包括 Web build outputs、Flutter registrants、Dart codegen、Go protobufs、Rust bindings，以及 Swift / Kotlin generated sources。
</Aside>

## Session 生命週期

| 事件 | 發生內容 |
|---|---|
| **SessionStart** | 建立 session 目錄並寫入 heartbeat |
| **UserPromptSubmit** | 追加 prompt、遞增 turn counter，並輪換 logs |
| **PreToolUse Edit/Write** | 同步擷取 pre-edit blob hash |
| **PostToolUse Edit/Write** | 擷取 post-edit blob hash 與檔案路徑 |
| **Stop** | 記錄 stop 事件（heartbeat 會保持有效 —— Stop = response 結束，不是 session 結束） |

<Aside>
  Git hooks 在注入 trailers 或記錄 notes 之前，會透過 heartbeat 驗證 session 是否仍然新鮮。heartbeat 會在 SessionStart 與每次 prompt 時更新。
</Aside>

<Aside type="tip">
  Agent Note 會識別目前的 agent。session metadata 會保存目前的 adapter，因此 `show`、`status` 與 commit recording 之後都能解析正確的 transcript format。
</Aside>

## Git Notes

Git notes 不會出現在 branch 清單、GitHub UI 或 CI 中，但依然可以 push 與 fetch。

```bash
# 與團隊共享
git push origin refs/notes/agentnote

# 從 remote 取回
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# 查看 note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` 會設定 auto-fetch，而 `pre-push` hook 會處理 auto-push。不需要手動命令。
</Aside>
