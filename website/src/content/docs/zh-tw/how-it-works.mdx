只要生成的 git hook 已安裝，大多數儲存庫裡就繼續正常使用 `git commit` 與 `git push` 即可。Cursor 也能從本機 transcript 還原 prompt / response，在 git hook 無法使用的環境下也能繼續追蹤 commit。

### 工作階段檔案

| 檔案 | 用途 |
|---|---|
| `prompts.jsonl` | 每行一個 prompt，並附帶 turn 編號 |
| `changes.jsonl` | AI 編輯過的檔案，以及 post-edit blob hashes |
| `pre_blobs.jsonl` | 每次 AI edit 前的檔案狀態 |
| `heartbeat` | 工作階段活動時間戳 |
| `turn` | 每次 prompt 都會遞增的單調計數器 |

## `📝 Context` 的收集方式

`📝 Context` 是只用於顯示的補充說明，用來幫助理解 `continue` 或 `做下一個` 這類很短的 prompt。它不會改變 attribution。

<img src="/AgentNote/images/context-dashboard-example.png" alt="Agent Note Dashboard showing Context before a short prompt" style="width: 100%; border-radius: 20px; border: 1px solid rgba(148, 163, 184, 0.18); box-shadow: 0 24px 80px rgba(15, 23, 42, 0.28);" />

- `reference` context 來自上一則 response。只有當上一則 response 提到最終 diff 中的 file path、file name 或 code identifier 時才會使用。
- `scope` context 來自目前的 response。只有當 response 開頭明確說明本次工作範圍時才會使用。
- Agent Note 不會只根據 `yes`、`continue`、`お願いします` 這類短確認來選擇 Context。如果沒有具體的 file 或 code 線索，就不會加入 Context。
- Context 永遠不會改變 `files_touched`、prompt ownership、AI ratio、attribution method，也不會影響是否保存 git note。

## Agent Note vs Entire

Agent Note 和 Entire 解決的是相關但不同的問題。

- Agent Note 會把 git commit 與產生該 commit 的 AI conversation 關聯起來。Entire 是更廣義的 checkpoint product，包含 rewind / resume workflow。
- Agent Note 會把永久記錄保存到 `refs/notes/agentnote`；暫時 session 檔案保留在 `.git/agentnote/` 下。Entire 以 checkpoint metadata 和 hosted web application 為中心。
- Agent Note 的 Dashboard 是以 git notes 為基礎的 optional static GitHub Pages。即使沒有 hosted service，也可以繼續使用 Agent Note。
- Agent Note 不嘗試還原舊的 workspace state。它專注於 reviewability：commit 為什麼改變、哪些 prompt 參與其中，以及 diff 中有多少是 AI-authored。

<Aside type="tip">
  prompt 清單來自 commit 到下一個 commit 之間的對話 window，而不是整個 session backlog。Agent Note 會保留能解釋目前 commit 的 planning、clarification 與 review prompt，同時裁掉被引用的舊 prompt 歷史，以及沒有保留到這個 commit 中的 edit turn。
</Aside>

<Aside>
  常見的 generated artifacts 會以 best-effort 的方式根據具體信號排除在 `AI ratio` 的分母之外，例如框架/快取路徑、生成的檔名/後綴，以及已提交的檔案 header。僅有 `build/`、`dist/`、`gen/`、`generated/`、`target/` 這類通用目錄名並不足以判定為 generated。如果你的 repository 需要提交 bundle 或 generated output，並希望它們繼續顯示但不影響 AI Ratio，請將路徑加入倉庫根目錄的 `.agentnoteignore` 檔案中。
</Aside>

## Session 生命週期

| 事件 | 發生內容 |
|---|---|
| **SessionStart** | 建立 session 目錄並寫入 heartbeat |
| **UserPromptSubmit** | 追加 prompt、遞增 turn counter，並輪換 logs |
| **PreToolUse Edit/Write** | 同步擷取 pre-edit blob hash |
| **PostToolUse Edit/Write** | 擷取 post-edit blob hash 與檔案路徑 |
| **Stop** | 記錄 stop 事件（heartbeat 會保持有效 —— Stop = response 結束，不是 session 結束） |

<Aside>
  Git hooks 在注入 trailers 或記錄 notes 之前，會透過 heartbeat 驗證 session 是否仍然新鮮，並確認存在可記錄的 session data。heartbeat 會在較長 turn 中由 hook events 更新；metadata-only session 會被略過。單獨的 transcript path 只是輔助 metadata，不是可記錄 data。
</Aside>

<Aside type="tip">
  Agent Note 會識別目前的 agent。session metadata 會保存目前的 adapter，因此 `show`、`status` 與 commit recording 之後都能解析正確的 transcript format。
</Aside>

## Git Notes

Git notes 不會出現在 branch 清單、GitHub UI 或 CI 中，但依然可以 push 與 fetch。

```bash
# 與團隊共享
git push origin refs/notes/agentnote

# 從 remote 取回
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# 查看 note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` 會設定 auto-fetch，而 `pre-push` hook 會處理 auto-push。不需要手動命令。
</Aside>
