## 快速設定

先在你的儲存庫裡執行一次：

```bash
npx agent-note init --agent claude
```

這樣就會替你建立 PR 報告 workflow：

```yaml
# .github/workflows/agentnote-pr-report.yml
name: Agent Note PR Report
on:
  pull_request:
    types: [opened, reopened, synchronize]
permissions:
  contents: read
  pull-requests: write
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0
      - uses: wasabeef/AgentNote@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

這樣就完成了。每個 PR 都會得到一份 AI 會話報告。

## Inputs

| Input | Default | 說明 |
|---|---|---|
| `base` | PR 的 base branch | 用來比較的分支 |
| `pr_output` | `description` | PR 報告的輸出位置：`description`、`comment` 或 `none` |
| `prompt_detail` | `standard` | PR 報告中 prompt 歷史的詳細程度：`compact`、`standard` 或 `full` |
| `dashboard` | `false` | 執行 Dashboard Mode，而不是 PR Report Mode。通常由 `agent-note init --dashboard` 產生 |

## Dashboard

```bash
npx agent-note init --agent claude --dashboard
```

<Aside type="tip">
  這也會額外加入 `.github/workflows/agentnote-dashboard.yml`，用來設定共享 Dashboard。
</Aside>

## Outputs

| Output | 類型 | 說明 |
|---|---|---|
| `overall_ai_ratio` | number | 整個 PR 的 AI ratio (0-100) |
| `overall_method` | string | `line`、`file`、`mixed` 或 `none` |
| `tracked_commits` | number | 帶有 agent-note 資料的 commit 數 |
| `total_commits` | number | PR 內的 commit 總數 |
| `total_prompts` | number | 所有 commit 的 prompt 總數 |
| `json` | string | 完整的結構化報告 |
| `markdown` | string | 已渲染的 Markdown 報告 |
| `should_deploy` | boolean string | Dashboard Mode 輸出，用來告訴呼叫方 workflow 是否需要發布 Pages |

## 報告內容

PR 報告包含：

- **Header** — `Total AI Ratio` 與進度條，以及下一行的 `Model`
- **表格** — 每個 commit 的 AI ratio、prompt 數與檔案列表
- **Prompts** — 可折疊的 `📝 Context`、`🧑 Prompt`、`🤖 Response` 區塊

<Aside type="tip">
  報告中的 commit hash 會直接連到 GitHub 上對應的 commit 頁面。
</Aside>
