This page explains the mechanics behind the report: what is captured locally, what becomes a git note, and how that data reaches PR Report and Dashboard. You do not need every detail to use Agent Note, but this is the place to check the model when something looks surprising.

With the generated git hooks installed, most repos can keep using plain `git commit` and `git push`. Cursor Preview can also recover prompt / response pairs from local transcripts, and its shell hooks can keep tracking commits when git hooks are unavailable.

## Storage

<CardGrid>
  <Card title="Layer 1: Local Temp" icon="laptop">
    Session data in `.git/agentnote/sessions/`. JSONL files for prompts, changes, and blob hashes. Rotated after each commit. Never pushed.
  </Card>
  <Card title="Layer 2: Git Notes" icon="document">
    Permanent record in `refs/notes/agentnote`. One JSON per commit. Pushable, fetchable, shareable with the team.
  </Card>
</CardGrid>

### Session Files

| File | Purpose |
|---|---|
| `prompts.jsonl` | One prompt per line with turn number |
| `changes.jsonl` | Files touched by AI with post-edit blob hashes |
| `pre_blobs.jsonl` | File state before each AI edit |
| `heartbeat` | Session activity timestamp |
| `turn` | Monotonic counter incremented on each prompt |

## How `📝 Context` Is Collected

`📝 Context` is a display-only note for short prompts such as "continue" or "do the next one". It helps readers understand the prompt without changing attribution.

<img src="/AgentNote/images/context-dashboard-example.png" alt="Agent Note Dashboard showing Context before a short prompt" style="width: 100%; border-radius: 20px; border: 1px solid rgba(148, 163, 184, 0.18); box-shadow: 0 24px 80px rgba(15, 23, 42, 0.28);" />

- `reference` context comes from the immediately previous response when it mentions a changed file path, file name, or code identifier from the final diff.
- `scope` context comes from the current response when its opening lines clearly name the work being done.
- Agent Note does not use approval words such as `yes`, `continue`, or `お願いします` as selection rules. If there is no concrete file or code clue, it skips Context.
- Context never changes `files_touched`, prompt ownership, AI ratio, attribution method, or whether a git note is saved.

## Agent Note vs Entire

Agent Note and Entire solve related but different problems.

- Agent Note links git commits to the AI conversation that produced them. Entire is a broader checkpoint product with rewind / resume workflows.
- Agent Note stores the permanent record in `refs/notes/agentnote`; temporary session files stay under `.git/agentnote/`. Entire centers checkpoint metadata and a hosted web application.
- Agent Note's Dashboard is optional static GitHub Pages backed by git notes. You can keep using Agent Note without a hosted service.
- Agent Note does not try to restore an old workspace state. It focuses on reviewability: why a commit changed, which prompts were involved, and how much of the diff was AI-authored.

<Aside type="tip">
  Prompt lists come from the commit-to-commit conversation window, not the whole session backlog. Agent Note keeps the useful planning, clarification, and review prompts for the current commit, while trimming quoted prompt-history blocks and edit turns that did not survive into this commit.
</Aside>

<Aside>
  Common generated artifacts are excluded from the AI ratio denominator on a best-effort basis using specific signals such as framework/cache paths, generated filenames/suffixes, and committed file headers. Generic directory names like `build/`, `dist/`, `gen/`, `generated/`, and `target/` are not enough by themselves.
</Aside>

## Session Lifecycle

| Event | What happens |
|---|---|
| **SessionStart** | Creates session directory, writes heartbeat |
| **UserPromptSubmit** | Appends prompt, increments turn counter, rotates logs |
| **PreToolUse Edit/Write** | Captures pre-edit blob hash (synchronous) |
| **PostToolUse Edit/Write** | Captures post-edit blob hash + file path |
| **Stop** | Logs stop event (heartbeat stays active — Stop = response end, not session end) |

<Aside>
  Git hooks verify session freshness via heartbeat before injecting trailers or recording notes. Heartbeat is updated on SessionStart and each prompt.
</Aside>

<Aside type="tip">
  Agent Note is agent-aware. Session metadata stores the active adapter, so `show`, `status`, and commit recording can resolve the right transcript format later.
</Aside>

## Git Notes

Git notes are invisible to branch listings, GitHub UI, and CI — but pushable and fetchable.

```bash
# Share with team
git push origin refs/notes/agentnote

# Fetch from remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# View a note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` configures auto-fetch and the `pre-push` hook handles auto-push. No manual commands needed.
</Aside>

For storage boundaries, visibility, and what is never uploaded to an Agent Note service, see [Data & Privacy](./data-and-privacy/).
