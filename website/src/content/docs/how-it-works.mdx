## Data Flow

<Steps>
1. You prompt your coding agent → hooks capture the prompt
2. The agent writes code → hooks or transcripts track files touched and attribution signals
3. You run `git commit` or `agent-note commit` → trailer injected, note recorded
4. You `git push` → notes auto-pushed to remote
</Steps>

No extra commands are needed when the repo-local git hooks are installed. Cursor preview also restores prompt / response pairs from response hooks or local transcripts, and its shell hooks provide a fallback path when git hooks are unavailable.

## Storage

<CardGrid>
  <Card title="Layer 1: Local Temp" icon="laptop">
    Session data in `.git/agentnote/sessions/`. JSONL files for prompts, changes, and blob hashes. Rotated after each commit. Never pushed.
  </Card>
  <Card title="Layer 2: Git Notes" icon="document">
    Permanent record in `refs/notes/agentnote`. One JSON per commit. Pushable, fetchable, shareable with the team.
  </Card>
</CardGrid>

### Session Files

| File | Purpose |
|---|---|
| `prompts.jsonl` | One prompt per line with turn number |
| `changes.jsonl` | Files touched by AI with post-edit blob hashes |
| `pre_blobs.jsonl` | File state before each AI edit |
| `heartbeat` | Session activity timestamp |
| `turn` | Monotonic counter incremented on each prompt |

### Note Schema

```json
{
  "v": 1,
  "agent": "claude",
  "session_id": "a1b2c3d4-...",
  "model": "claude-sonnet-4-20250514",

  ],

    "ai_ratio": 73,
    "method": "line",
    "lines": { "ai_added": 146, "total_added": 200, "deleted": 12 }
  }
}
```

## Attribution Modes

Claude Code computes AI authorship per-line using a 3-diff position algorithm:

| Diff | From | To | What it captures |
|---|---|---|---|
| diff1 | parent commit | committed file | All changes in this commit |
| diff2 | pre-AI-edit | committed file | AI edit + human edits after |
| diff3 | post-AI-edit | committed file | Human edits only |

**AI positions = diff2 positions NOT in diff3**

All three diffs target the committed blob, so positions are directly comparable.

<Aside type="tip">
  Deletions are tracked separately — not attributed to AI or human. Falls back to file-level binary attribution when blob data is unavailable.
</Aside>

Codex CLI uses transcript-driven attribution. When transcript patch counts match the final commit diff, Agent Note upgrades Codex commits to `method: "line"`. Otherwise it falls back to `method: "file"` to avoid false positives. Shell-only edits that never surface as transcript patch data stay on the safe side: Agent Note can still preserve the prompt / response pair and observed Codex tools, but it does not guess `files_touched` or AI-authored files from shell steps alone. Those commits therefore stay at `method: "file"` with `0%` AI attribution unless another primary signal ties files to the agent. If the transcript cannot be read at commit time, Agent Note warns and skips note creation rather than recording uncertain attribution data.

Cursor preview uses `afterFileEdit` and `afterTabFileEdit` hooks for attribution. When their edit counts match and the final committed blob still matches the last AI edit, Agent Note safely upgrades those files to `method: "line"`. Otherwise it falls back to `method: "file"`, while still recovering prompt / response pairs from Cursor response hooks or local transcripts when they are available.

Gemini CLI preview uses `BeforeTool`/`AfterTool` hooks on `write_file` and `replace` tools for file-level attribution. Commit detection uses the pending-commit pattern via `BeforeTool`/`AfterTool` on `run_shell_command`. Prompt and response recovery comes from `BeforeAgent`/`AfterAgent` hooks and JSONL transcript parsing.

<Aside type="tip">
  Prompt lists come from the causal conversation around the final diff, not the whole session backlog. Agent Note also excludes common generated artifacts from the AI ratio denominator on a best-effort basis, including Web build outputs, Flutter registrants, Dart codegen, Go protobufs, Rust bindings, and Swift / Kotlin generated sources.
</Aside>

## Session Lifecycle

| Event | What happens |
|---|---|
| **SessionStart** | Creates session directory, writes heartbeat |
| **UserPromptSubmit** | Appends prompt, increments turn counter, rotates logs |
| **PreToolUse Edit/Write** | Captures pre-edit blob hash (synchronous) |
| **PostToolUse Edit/Write** | Captures post-edit blob hash + file path |
| **Stop** | Logs stop event (heartbeat stays active — Stop = response end, not session end) |

<Aside>
  Git hooks verify session freshness via heartbeat before injecting trailers or recording notes. Heartbeat is updated on SessionStart and each prompt.
</Aside>

<Aside type="tip">
  Agent Note is agent-aware. Session metadata stores the active adapter, so `show`, `status`, and commit recording can resolve the right transcript format later.
</Aside>

## Git Notes

Git notes are invisible to branch listings, GitHub UI, and CI — but pushable and fetchable.

```bash
# Share with team
git push origin refs/notes/agentnote

# Fetch from remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# View a note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` configures auto-fetch and the `pre-push` hook handles auto-push. No manual commands needed.
</Aside>
