## Common Tasks

- First setup: `agent-note init`
- Check what a commit recorded: `agent-note show`
- Ask why a line changed: `agent-note why`
- Review recent activity: `agent-note log`
- Generate a PR Report: `agent-note pr`
- Verify hooks and the active adapter: `agent-note status`

## agent-note init

Set up Agent Note in the current repository.

```bash
npx agent-note init --agent claude
```

| Flag | Description |
|---|---|
| `--agent <name...>` | **Required.** Choose one or more agent adapters: `claude`, `codex`, `cursor`, or `gemini` |
| `--dashboard` | Also create `.github/workflows/agentnote-dashboard.yml` for the shared Dashboard |
| `--no-hooks` | Skip agent hook configuration |
| `--no-git-hooks` | Skip git hook installation |
| `--no-action` | Skip workflow creation |
| `--no-notes` | Skip auto-fetch config |
| `--hooks` | Only install agent hooks |
| `--action` | Only create workflow |

## agent-note show

Show AI session details for a commit.

```bash
npx agent-note show          # HEAD
npx agent-note show abc1234  # commit SHA
```

```

session: a1b2c3d4-5678-90ab-cdef-111122223333

ai:      60% (45/75 lines) [█████░░░]
model:   claude-sonnet-4-20250514
agent:   claude
files:   5 changed, 3 by AI

  src/middleware/auth.ts  🤖
  src/types/token.ts  🤖
  CHANGELOG.md  👤

prompts: 2

  1. Implement JWT auth middleware with refresh token rotation
  2. Add tests for expired token and invalid signature
```

## agent-note why

Show the Agent Note context behind the commit that last changed a file line.

```bash
npx agent-note why src/middleware/auth.ts:42
npx agent-note why src/middleware/auth.ts:42-55
```

<Aside>
  `why` shortens the manual `git blame` → `git notes --ref=agentnote show <commit>` lookup flow. It shows commit-level or file-level prompt evidence and stays explicit that exact line-to-prompt attribution is not stored yet.
</Aside>

## agent-note log

List recent commits with AI ratio.

```bash
npx agent-note log      # last 10
npx agent-note log 20   # last 20
```

```
ce941f7 feat: add auth  [a1b2c3d4… | 🤖60% | 2p]
326a568 test: add tests [a1b2c3d4… | 🤖100% | 1p]
ba091be fix: update deps
```

## agent-note pr

Generate a PR Report.

<Tabs>
  <TabItem label="Markdown">
  ```bash
  npx agent-note pr origin/main
  ```
  </TabItem>
  <TabItem label="Detail">
  ```bash
  npx agent-note pr origin/main --prompt-detail compact
  ```
  </TabItem>
  <TabItem label="JSON">
  ```bash
  npx agent-note pr origin/main --json
  ```
  </TabItem>
  <TabItem label="Update PR">
  ```bash
  # Insert into PR description
  npx agent-note pr origin/main --output description --update 42

  # Post as PR comment
  npx agent-note pr origin/main --output comment --update 42
  ```
  </TabItem>
</Tabs>

<Aside>
  The default prompt detail is `compact`: it keeps the PR report focused by showing the prompts that explain the commit, while `full` shows every stored prompt.
</Aside>

## agent-note session

Show all commits belonging to a session.

```bash
npx agent-note session a1b2c3d4-5678-90ab-cdef-111122223333
```

## agent-note status

Show current tracking state.

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

<Aside>
  `agent:` shows which adapters are enabled, `capture:` summarizes what the active agent hooks collect, `git:` shows whether the managed repository-local git hooks are installed, and `commit:` shows the primary tracking path. If `commit:` shows fallback mode, prefer `agent-note commit`.
</Aside>

## agent-note deinit

Remove Agent Note hooks and configuration from the current repository.

```bash
npx agent-note deinit --agent claude
```

| Flag | Description |
|---|---|
| `--agent <name...>` | **Required.** Remove one or more agents: `claude`, `codex`, `cursor`, or `gemini` |
| `--remove-workflow` | Also delete `.github/workflows/agentnote-pr-report.yml` and `.github/workflows/agentnote-dashboard.yml` |
| `--keep-notes` | Keep the notes auto-fetch configuration |

<Aside>
  Shared infrastructure (git hooks, notes config) is preserved when other agents are still enabled.
</Aside>

## agent-note commit

Convenience wrapper for `git commit` with session trailer and note recording.

```bash
npx agent-note commit -m "feat: add auth"
```

<Aside type="tip">
  Not needed if git hooks are installed — `git commit` does the same thing automatically.
</Aside>

<Aside>
  Cursor preview tracks plain `git commit` normally when the generated git hooks are installed.
  Its hooks also capture prompts, response metadata, and `afterFileEdit` / `afterTabFileEdit` attribution data, with a safe line-level upgrade when edit counts match and the committed file still matches the last AI edit. `agent-note commit` remains a useful fallback wrapper when git hooks are unavailable.
</Aside>
