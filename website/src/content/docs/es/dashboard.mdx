## Qué obtienes

El Dashboard le da a tu equipo una única URL compartida para el historial de Agent Note.

- Historial de PR abiertas y merged en un solo lugar
- Commits, prompts y attribution sin abrir cada PR por separado
- Una única URL de GitHub Pages, normalmente bajo `/dashboard/`

Úsalo cuando el PR report ya no sea suficiente y quieras una vista compartida para el equipo.

## Antes de empezar

Solo necesitas el Dashboard si quieres una vista web compartida.

- Si los informes de sesión de IA dentro de las pull requests te alcanzan, quédate con [GitHub Action](./github-action/).
- Si quieres una vista compartida en Pages para historial abierto y merged, sigue aquí.

## Configuración rápida

1. Run Dashboard setup once.

```bash
npx agent-note init --agent claude --dashboard
```

2. Commit both workflow files.

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. In GitHub Pages, choose **GitHub Actions** as the source.

The generated Dashboard workflow restores data from `gh-pages`, refreshes it from `refs/notes/agentnote`, and publishes the shared `/dashboard/` view.

## URL pública

Normalmente el Dashboard se publica aquí:

- `https://<owner>.github.io/<repo>/dashboard/`

Por ejemplo:

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Si publicas con GitHub Pages, tu workflow de Pages pasa los valores específicos del repo para `SITE` y `BASE` durante el build. En la mayoría de los repositorios puedes usar el workflow de ejemplo sin tocar el Dashboard source.
</Aside>

## Primer deploy

Incluso en un repo nuevo, el Dashboard puede acumular datos antes del primer deploy de producción. La URL pública de Pages aparece recién después del primer deploy a `default branch`.

## Cómo funciona

- `refs/notes/agentnote` remains the source of truth
- The generated Dashboard workflow writes shared snapshots to `gh-pages/dashboard/notes/*.json`
- GitHub Pages serves the built Dashboard at `/dashboard/`

The repository does not commit sample Dashboard JSON. A fresh Dashboard stays empty until the workflow restores or generates note files. `pull_request` refreshes open history, and `default branch` publishes the merged state.
