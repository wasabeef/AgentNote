Esta página muestra los comandos que los usuarios ejecutan directamente. Para el primer setup, empieza por [Instalación](./installation/).

## Tareas comunes

- Primer setup: `agent-note init`
- Quitar Agent Note del repositorio: `agent-note deinit`
- Verificar el estado: `agent-note status`
- Revisar actividad reciente: `agent-note log`
- Mostrar los datos de un commit: `agent-note show`
- Entender por qué cambió una línea: `agent-note why`

<Aside>
  PR Report y Dashboard son funciones de GitHub Action. `agent-note init` normalmente crea los archivos de workflow; los reportes y el Dashboard se generan después cuando esos workflows se ejecutan.
</Aside>

## agent-note init

Configura Agent Note en el repositorio.

```bash
npx agent-note init --agent claude
```

| Flag | Descripción |
|---|---|
| `--agent <name...>` | Elige uno o más adapters de agent: `claude`, `codex`, `cursor` or `gemini` |
| `--dashboard` | También crea el workflow del Dashboard compartido |
| `--no-hooks` | Omite la configuración de hooks del agent |
| `--no-git-hooks` | Omite la instalación de Git Hooks |
| `--no-action` | Omite la creación de workflows |
| `--no-notes` | Omite la configuración de notes auto-fetch |
| `--hooks` | Solo instala hooks del agent |
| `--action` | Solo crea workflows |

## agent-note deinit

Elimina hooks y configuración de Agent Note para uno o más agents.

```bash
npx agent-note deinit --agent claude
```

| Flag | Descripción |
|---|---|
| `--agent <name...>` | `claude`, `codex`, `cursor` or `gemini` |
| `--remove-workflow` | También elimina los workflows de PR Report y Dashboard |
| `--keep-notes` | Conserva la configuración de notes auto-fetch |

## agent-note status

Muestra el estado de tracking.

```bash
npx agent-note status
```

```text

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

## agent-note log

Lista commits recientes con AI Ratio.

```bash
npx agent-note log
npx agent-note log 20
```

```text
ce941f7 feat: add auth  [a1b2c3d4… | 🤖60% | 2p]
326a568 test: add tests [a1b2c3d4… | 🤖100% | 1p]
ba091be fix: update deps
```

## agent-note show

Muestra la sesión AI de un commit.

```bash
npx agent-note show
npx agent-note show abc1234
```

```text

session: a1b2c3d4-5678-4abc-8def-111122223333
```

## agent-note why

Muestra el contexto de Agent Note detrás de una línea modificada.

```bash
npx agent-note why src/middleware/auth.ts:42
npx agent-note why src/middleware/auth.ts:42-55
npx agent-note why src/middleware/auth.ts:42:7
npx agent-note why src/middleware/auth.ts#L42
npx agent-note why src/middleware/auth.ts#L42-L55
npx agent-note why src/middleware/auth.ts#L42C7
npx agent-note why @src/middleware/auth.ts#L42
npx agent-note why https://github.com/owner/repo/blob/main/src/middleware/auth.ts#L42
npx agent-note why file:///workspace/repo/src/middleware/auth.ts#L42
npx agent-note why vscode://file/workspace/repo/src/middleware/auth.ts:42:7
```

<Aside>
  `why` acepta `path:line`, `path#Lline`, GitHub file URLs, `file://...`, `vscode://file...` y paths con `@` inicial copiados de un AI Agent.
</Aside>

## AI Agent Skill

Instala el Skill de Agent Note para pedir estas tareas en lenguaje natural.

```bash
gh skill install wasabeef/AgentNote agent-note --agent codex --scope user
```

Para `gh skill install`, elige el identificador de agent adecuado: `codex`, `claude-code`, `cursor` or `gemini-cli`.
