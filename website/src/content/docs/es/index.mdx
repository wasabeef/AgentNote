## Qué hace

<CardGrid stagger>
  <Card title="Mantener un registro de las líneas escritas por IA" icon="pencil">
    Más fino que `git blame`. Claude Code es completo a nivel de línea; otros agentes se actualizan cuando las condiciones coinciden.
  </Card>
  <Card title="Prompts y respuestas guardados junto al cambio" icon="document">
    Nombre del modelo y uso de herramientas también — todo empaquetado en git notes con el cambio.
  </Card>
  <Card title="Reportes automáticos en PR" icon="rocket">
    Proporción de IA, cantidad de prompts, hashes de commits. Los revisores ven el contexto antes del código.
  </Card>
  <Card title="Escrito solo en git notes" icon="setting">
    Las ramas se mantienen limpias. Push / fetch para compartir con el equipo y Un Dashboard estático opcional en GitHub Pages.
  </Card>
</CardGrid>

## Configuración

<Steps>
1. Ejecuta init en tu repositorio

   ```bash
   npx agent-note init --agent claude
   ```

   - Para Codex CLI: `npx agent-note init --agent codex`
   - Para Cursor: `npx agent-note init --agent cursor`
   - Para Gemini CLI: `npx agent-note init --agent gemini`
   - Varios agentes a la vez: `npx agent-note init --agent claude --agent cursor`

2. Haz commit de los archivos generados

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   - Codex CLI: versionar `.codex/config.toml` y `.codex/hooks.json`
   - Cursor: versionar `.cursor/hooks.json`
   - Gemini CLI: versionar `.gemini/settings.json`

3. Codifica con IA.
   Con los git hooks generados instalados, basta con usar `git commit` normalmente.
   `agent-note commit -m "..."` queda como alternativa cuando los git hooks no están disponibles.
</Steps>

## Verifica tu configuración

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` muestra el adaptador activo, `capture:` resume lo que recopilan los hooks activos, `git:` muestra si los git hooks administrados están instalados, y `commit:` indica si el `git commit` normal es la ruta principal.
