Agent Note guarda el mismo formato de git note para todos los agents, pero cada agent expone hooks y transcripts distintos. Esta página resume qué esperar antes de activarlo para un equipo.

## Capability Matrix

| Agent | Estado | Prompt | Response | Files | AI Ratio | Line Estimate |
|---|---|---|---|---|---|---|
| Claude Code | Full support | Sí | Sí | Sí | Sí | Por defecto |
| Codex CLI | Supported | Sí | Sí | Sí | Sí | Cuando el historial de parches de Codex coincide con el Commit final |
| Cursor | Supported | Sí | Sí | Sí | Sí | Cuando los conteos de edit y el blob final coinciden |
| Gemini CLI | Preview | Sí | Sí | Sí | Sí | Todavía no |

<Aside>
  Files significa que Agent Note puede mostrar qué archivos commiteados tocó el Agent. Line Estimate significa que también puede estimar líneas escritas por la AI en lugar de contar solo archivos. Preview significa que Agent Note sigue el comportamiento público actual, pero el formato local del agent puede cambiar.
</Aside>

## Setup recomendado

```bash
npx agent-note init --agent claude
```

Activa solo los agents que tu equipo usa. Puedes activar varios en el mismo repository.

```bash
npx agent-note init --agent claude codex cursor gemini
```

Cada developer debe ejecutar `init` una vez después de clonar, porque los git hooks viven dentro de `.git/`.

## Elegir según lo que esperas

| Necesitas... | Espera... |
|---|---|
| La atribución por línea más completa hoy | Claude Code |
| Soporte de Codex CLI con atribución conservadora | Codex CLI |
| Historial del editor Cursor y recuperación de respuestas | Cursor |
| Captura de prompt / response con Gemini CLI | Gemini CLI |

## Attribution

Line-level cuenta líneas añadidas por IA cuando Agent Note puede comparar el edit de IA, los edits humanos posteriores y el diff final. File-level se usa cuando esa prueba no es suficiente.

Agent Note prefiere un fallback conservador antes que adivinar.

## Notas por agent

### Claude Code

Claude Code entrega prompts, responses y edits por hooks. Agent Note guarda el estado del archivo antes y después del edit de IA y resta los edits humanos posteriores.

### Codex CLI

Codex CLI es transcript-driven. Agent Note lee transcripts locales y sube a line-level solo cuando los conteos `apply_patch` coinciden con el diff final.

### Cursor

Cursor usa `afterFileEdit` / `afterTabFileEdit`, además de response hooks o transcripts locales. Line-level solo es posible si el blob final sigue coincidiendo con el último edit de IA.

### Gemini CLI

Gemini CLI usa `BeforeTool` / `AfterTool` para archivos y `BeforeAgent` / `AfterAgent` para prompt / response. La attribution actual es file-level.

## Datos faltantes

- No se escribe note para commits sin archivos AI-authored.
- Un prompt antiguo puede omitirse si pertenece a otra ventana de commit.
- Una response puede faltar si el agent no la expone.
- Los generated artifacts y los paths listados en la `.agentnoteignore` en la raíz del repository siguen visibles en Notes, PR Report y Dashboard; solo se excluyen del denominador del AI ratio.

Para investigar, empieza por [Troubleshooting](./troubleshooting/).
