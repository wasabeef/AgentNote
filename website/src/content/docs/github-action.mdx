## Quick Setup

Add this workflow — created automatically by `agent-note init`:

```yaml
# .github/workflows/agentnote.yml
name: Agent Note
on:
  pull_request:
    types: [opened, synchronize]
permissions:
  contents: read
  pull-requests: write
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: wasabeef/AgentNote@v0
```

That's it. Every PR gets an AI session report.

## Inputs

| Input | Default | Description |
|---|---|---|
| `base` | PR base branch | Base branch to compare against |
| `pr_output` | `description` | PR report destination: `description`, `comment`, or `none` |
| `dashboard` | `false` | Write/update dashboard snapshot files for the current PR |
| `dashboard_dir` | `packages/dashboard/public/view-demo` | Target directory for `notes/*.json` |
| `output` | — | Legacy alias for `pr_output` (`description` or `comment`) |
| `comment` | `true` | Legacy: set to `false` to disable PR report posting |

### Post as comment instead of description

```yaml
- uses: wasabeef/AgentNote@v0
  with:
    pr_output: comment
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

### Generate dashboard data alongside the PR report

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
    dashboard_dir: packages/dashboard/public/view-demo
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## Outputs

| Output | Type | Description |
|---|---|---|
| `overall_ai_ratio` | number | PR-wide AI ratio (0-100) |
| `overall_method` | string | `line`, `file`, `mixed`, or `none` |
| `tracked_commits` | number | Commits with agent-note data |
| `total_commits` | number | Total commits in PR |
| `total_prompts` | number | Total prompts across all commits |
| `json` | string | Full structured report |
| `markdown` | string | Rendered markdown report |
| `dashboard_dir` | string | Directory where the dashboard bundle was written |
| `dashboard_commits` | number | Number of tracked commits written to the dashboard bundle |

### Use outputs in workflow

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    base: main

- run: echo "AI ratio: ${{ steps.agent-note.outputs.overall_ai_ratio }}%"
```

## Report Format

The PR report includes:

- **Header** — AI ratio with progress bar, line counts, commit count, prompts, model
- **Table** — per-commit AI ratio, line counts, prompts, files with attribution
- **Prompts** — collapsible section with per-commit prompt/response pairs

<Aside type="tip">
  Commit hashes in the report are linked to their GitHub commit page.
</Aside>

## Notes Sharing

The action fetches `refs/notes/agentnote` from the remote. The `pre-push` git hook installed by `agent-note init` auto-pushes notes on every `git push` — no manual push needed.

## Dashboard Package

The interactive dashboard now lives in `packages/dashboard/` and is deployed alongside the docs site under `/dashboard/`. The action can update the bundle files in that package, while your GitHub Pages workflow decides when to publish them.

## GitHub Pages Sync

For the live dashboard, prefer a `push`-driven Pages workflow over `pull_request`-only updates.

- `push` on a PR branch captures commit notes before squash merge can hide those SHAs from `main`
- `gh-pages/dashboard/notes/*.json` is the persistent data store
- the dashboard app itself can stay on the current `main` shell while only the note data changes on branch pushes
- `push` on `main` finalizes matching PR notes as `merged`

Recommended flow:

1. Restore `dashboard/notes` from the `gh-pages` branch.
2. On branch pushes, resolve the open PR for `owner:branch`, remove existing notes for that PR number, then rewrite only the PR's current commits.
3. On `main` pushes, update matching PR notes to `merged` and write any direct-push notes.
4. Build the Pages artifact and publish it.

This keeps `main` free of generated JSON while still letting one shared dashboard show open and merged PR history.
