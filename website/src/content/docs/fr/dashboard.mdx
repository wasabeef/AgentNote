## Ce que vous obtenez

Le Dashboard donne à l’équipe une URL unique pour consulter l’historique Agent Note.

- Les PR ouvertes et mergées au même endroit
- Les commits, prompts et données d’attribution sans ouvrir chaque PR
- Une seule URL GitHub Pages, généralement sous `/dashboard/`

Utilisez-le quand le rapport de PR ne suffit plus et qu’il vous faut une vue partagée pour l’équipe.

## Avant de commencer

Vous n’avez besoin du Dashboard que si vous voulez une vue web partagée.

- Si les rapports de session IA dans les pull requests vous suffisent, restez sur [GitHub Action](./github-action/).
- Si vous voulez une vue Pages partagée pour l’historique ouvert et mergé, continuez ici.

## Mise en place rapide

1. Run Dashboard setup once.

```bash
npx agent-note init --agent claude --dashboard
```

2. Commit both workflow files.

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. In GitHub Pages, choose **GitHub Actions** as the source.

The generated Dashboard workflow restores data from `gh-pages`, refreshes it from `refs/notes/agentnote`, and publishes the shared `/dashboard/` view.

## URL publique

Le Dashboard est généralement publié ici :

- `https://<owner>.github.io/<repo>/dashboard/`

Par exemple :

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Si vous publiez avec GitHub Pages, votre workflow Pages transmet les valeurs spécifiques au dépôt pour `SITE` et `BASE` au moment du build. Dans la plupart des dépôts, le workflow d’exemple suffit sans modification de la source du Dashboard.
</Aside>

## Premier déploiement

Même un dépôt tout neuf peut accumuler des données de Dashboard avant le premier déploiement de production. L’URL publique Pages n’apparaît toutefois qu’après le premier déploiement sur `default branch`.

## Comment ça fonctionne

- `refs/notes/agentnote` remains the source of truth
- The generated Dashboard workflow writes shared snapshots to `gh-pages/dashboard/notes/*.json`
- GitHub Pages serves the built Dashboard at `/dashboard/`

The repository does not commit sample Dashboard JSON. A fresh Dashboard stays empty until the workflow restores or generates note files. `pull_request` refreshes open history, and `default branch` publishes the merged state.
