## Ce que vous obtenez

Le Dashboard donne à l’équipe une URL unique pour consulter l’historique Agent Note.

- Les PR ouvertes et mergées au même endroit
- Les commits, prompts et données d’attribution sans ouvrir chaque PR
- Une seule URL GitHub Pages, généralement sous `/dashboard/`

Utilisez-le quand le rapport de PR ne suffit plus et qu’il vous faut une vue partagée pour l’équipe.

## Avant de commencer

Vous n’avez besoin du Dashboard que si vous voulez une vue web partagée.

- Si les rapports de session IA dans les pull requests vous suffisent, restez sur [GitHub Action](./github-action/).
- Si vous voulez une vue Pages partagée pour l’historique ouvert et mergé, continuez ici.

## Mise en place rapide

Commencez par laisser l’Action écrire les fichiers du Dashboard pour chaque PR :

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Cela crée ou met à jour `packages/dashboard/public/notes/*.json` pour la PR en cours. Ensuite, votre workflow Pages restaure, enregistre et publie ces fichiers.

## URL publique

Le Dashboard est généralement publié ici :

- `https://<owner>.github.io/<repo>/dashboard/`

Par exemple :

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Si vous publiez avec GitHub Pages, votre workflow Pages transmet les valeurs spécifiques au dépôt pour `SITE` et `BASE` au moment du build. Dans la plupart des dépôts, le workflow d’exemple suffit sans modification de `packages/dashboard`.
</Aside>

## Premier déploiement

Même un dépôt tout neuf peut accumuler des données de Dashboard avant le premier déploiement de production. L’URL publique Pages n’apparaît toutefois qu’après le premier déploiement sur `main`.

## Comment ça fonctionne

- `packages/dashboard/` est l’application Astro qui affiche le Dashboard
- `packages/dashboard/public/notes/*.json` est le bundle lu au moment du build
- `gh-pages/dashboard/notes/*.json` est le stockage durable de l’historique partagé

Le dépôt ne contient pas de JSON d’exemple pour le Dashboard. Un Dashboard neuf reste vide jusqu’à ce que votre workflow restaure ou génère les fichiers de notes. Dans beaucoup d’équipes, les notes de PR sont mises à jour sur `pull_request`, puis l’état mergé est publié depuis `main`.
