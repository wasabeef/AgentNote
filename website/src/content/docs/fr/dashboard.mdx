## Ce que vous obtenez

Le Dashboard donne à l’équipe une URL unique pour consulter l’historique Agent Note.

- Les PR ouvertes et mergées au même endroit
- Les commits, prompts et données d’attribution sans ouvrir chaque PR
- Une seule URL GitHub Pages, généralement sous `/dashboard/`

Utilisez-le quand le rapport de PR ne suffit plus et qu’il vous faut une vue partagée pour l’équipe.

## Avant de commencer

Vous n’avez besoin du Dashboard que si vous voulez une vue web partagée.

- Si les rapports de session IA dans les pull requests vous suffisent, restez sur [GitHub Action](./github-action/).
- Si vous voulez une vue Pages partagée pour l’historique ouvert et mergé, continuez ici.

## Mise en place rapide

1. Lancez une fois la configuration du Dashboard.

```bash
npx agent-note init --agent claude --dashboard
```

2. Committez les deux fichiers de workflow.

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. Dans GitHub Pages, choisissez **GitHub Actions** comme source.

Le workflow Dashboard généré utilise `wasabeef/AgentNote@v0` avec `dashboard: true` pour restaurer, synchroniser, builder, uploader l'artefact et persister les notes, puis publie la vue partagée sous `/dashboard/`.

## URL publique

Le Dashboard est généralement publié ici :

- `https://<owner>.github.io/<repo>/dashboard/`

Par exemple :

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Si vous publiez avec GitHub Pages, votre workflow Pages transmet les valeurs spécifiques au dépôt pour `SITE` et `BASE` au moment du build. Dans la plupart des dépôts, le workflow d’exemple suffit sans modification de la source du Dashboard.
</Aside>

## Premier déploiement

Même un dépôt tout neuf peut accumuler des données de Dashboard avant le premier déploiement de production. Si les déploiements de PR sont autorisés, l’URL publique Pages peut apparaître après le premier run `pull_request` réussi. Sinon, elle apparaît après le premier déploiement réussi depuis `default branch`.

## Comment ça fonctionne

- `refs/notes/agentnote` reste la Source of Truth
- `wasabeef/AgentNote@v0` en mode Dashboard écrit les snapshots de notes dans `gh-pages/dashboard/notes/*.json`
- GitHub Pages sert le Dashboard généré sous `/dashboard/`

Le dépôt ne commit pas d’exemples JSON pour le Dashboard. Un Dashboard neuf reste vide tant que le workflow ne restaure ni ne génère les note files. `pull_request` publie l’état Open, puis les pushes vers `default branch` le remplacent par l’état Merged.
