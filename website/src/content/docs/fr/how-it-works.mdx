Une fois les git hooks générés installés, la plupart des dépôts peuvent continuer à utiliser simplement `git commit` et `git push`. Cursor Preview récupère aussi les paires prompt / response depuis les transcripts locaux et peut continuer à suivre les commits quand les git hooks ne sont pas disponibles.

### Fichiers de session

| Fichier | Rôle |
|---|---|
| `prompts.jsonl` | Un prompt par ligne avec numéro de turn |
| `changes.jsonl` | Fichiers touchés par l’IA avec blob hashes post-édition |
| `pre_blobs.jsonl` | État du fichier avant chaque édition IA |
| `heartbeat` | Horodatage d’activité de la session |
| `turn` | Compteur monotone incrémenté à chaque prompt |

<Aside type="tip">
  Les listes de prompts viennent de la fenêtre de conversation entre deux commits, pas de tout l’historique de la session. Agent Note garde les prompts utiles de planification, de clarification et de revue pour le commit courant, et retire l’historique de prompts cité ainsi que les tours d’édition qui ne survivent pas dans ce commit. Si un prompt court dépend de la réponse précédente mais que ce turn appartient déjà à un commit plus ancien, Agent Note peut ajouter un champ `context` uniquement pour l’affichage avant le prompt; cela ne change jamais l’attribution.
</Aside>

<Aside>
  Les generated artifacts courants sont exclus du dénominateur de `AI ratio` en best effort à partir de signaux précis, comme les chemins de framework/cache, les noms ou suffixes générés et les en-têtes des fichiers commités. Les noms de dossiers génériques comme `build/`, `dist/`, `gen/`, `generated/` et `target/` ne suffisent pas à eux seuls.
</Aside>

## Cycle de vie de la session

| Événement | Ce qui se passe |
|---|---|
| **SessionStart** | Crée le répertoire de session et écrit le heartbeat |
| **UserPromptSubmit** | Ajoute le prompt, incrémente le compteur de turn et fait tourner les logs |
| **PreToolUse Edit/Write** | Capture le pre-edit blob hash de façon synchrone |
| **PostToolUse Edit/Write** | Capture le post-edit blob hash et le chemin du fichier |
| **Stop** | Journalise l’événement stop (le heartbeat reste actif — Stop = fin de réponse, pas fin de session) |

<Aside>
  Les git hooks vérifient la fraîcheur de la session via le heartbeat avant d’injecter les trailers ou d’enregistrer les notes. Le heartbeat est mis à jour à SessionStart et à chaque prompt.
</Aside>

<Aside type="tip">
  Agent Note garde la trace de l’agent qui a produit la session. Les métadonnées de session stockent l’adapter actif afin que `show`, `status` et l’enregistrement du commit puissent retrouver plus tard le bon format de transcript.
</Aside>

## Git Notes

Les git notes sont invisibles dans les listes de branches, l’interface GitHub et CI — mais elles peuvent être poussées et récupérées.

```bash
# Partager avec l’équipe
git push origin refs/notes/agentnote

# Récupérer depuis le remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# Afficher une note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` configure l’auto-fetch, et le hook `pre-push` gère l’auto-push. Aucune commande manuelle n’est nécessaire.
</Aside>
