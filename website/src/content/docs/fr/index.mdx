## Ce que fait Agent Note

<CardGrid stagger>
  <Card title="Attribution IA ligne par ligne" icon="pencil">
    Claude Code bénéficie d'une attribution complète ligne par ligne. Codex passe au niveau ligne quand les compteurs de patch du transcript correspondent au commit final.
  </Card>
  <Card title="Suivi de session" icon="document">
    Prompts, réponses, nom du modèle et outils utilisés — le tout lié aux commits qu'ils ont produits.
  </Card>
  <Card title="Rapports de PR" icon="rocket">
    Rapports automatiques de session IA sur les pull requests avec ratio par commit, nombre de lignes et hashes liés.
  </Card>
  <Card title="Natif Git" icon="setting">
    Stocké sous forme de git notes — invisible pour les branches et la CI. Peut être poussé, récupéré et partagé.
  </Card>
</CardGrid>

## Configuration en 60 secondes

<Steps>
1. Lancez init dans votre dépôt

   ```bash
   npx @wasabeef/agentnote init
   ```

   Pour Codex CLI :

   ```bash
   npx @wasabeef/agentnote init --agent codex
   ```

   Pour Cursor :

   ```bash
   npx @wasabeef/agentnote init --agent cursor
   ```

   Pour Gemini CLI :

   ```bash
   npx @wasabeef/agentnote init --agent gemini
   ```

2. Committez les fichiers générés

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agentnote"
   git push
   ```

   Les dépôts Codex versionnent `.codex/config.toml` et `.codex/hooks.json` à la place de `.claude/settings.json`.
   Les dépôts Cursor versionnent `.cursor/hooks.json`.
   Les dépôts Gemini CLI commitent `.gemini/settings.json`.

3. Claude Code et Codex peuvent utiliser `git commit` normalement.
   Avec les git hooks générés, Cursor preview peut aussi utiliser `git commit` normalement.
   Utilisez `agentnote commit -m "..."` comme solution de secours si les git hooks ne sont pas disponibles.
</Steps>

## Compatibilité

| Agent | Statut | Notes |
|---|---|---|
| Claude Code | Support complet | Attribution ligne par ligne via des paires de blobs capturées par hook |
| Codex CLI | Preview | Basé sur le transcript ; passe au niveau ligne quand les compteurs de patch correspondent au commit |
| Cursor | Preview | Attribution via `afterFileEdit` / `afterTabFileEdit`, avec passage sûr au niveau ligne quand le blob final correspond encore à l'édition IA |
| Gemini CLI | Preview | Attribution par fichier via les hooks `BeforeTool`/`AfterTool` dans `.gemini/settings.json` |
