## Ce que ça fait

<CardGrid stagger>
  <Card title="Garder une trace des lignes écrites par l'IA" icon="pencil">
    Plus fin que `git blame`. Claude Code est complet au niveau ligne ; les autres agents sont mis à niveau quand les conditions correspondent.
  </Card>
  <Card title="Prompts et réponses sauvegardés ensemble" icon="document">
    Le nom du modèle et l'utilisation des outils aussi — tout regroupé dans les git notes avec le changement.
  </Card>
  <Card title="Rapports PR automatiques" icon="rocket">
    Ratio IA, nombre de prompts, hashes de commits. Les relecteurs voient le contexte avant le code.
  </Card>
  <Card title="Écrit uniquement dans les git notes" icon="setting">
    Les branches restent propres. Push / fetch pour partager avec l'équipe, avec un Dashboard statique optionnel sur GitHub Pages.
  </Card>
</CardGrid>

## Configuration

<Steps>
1. Lancez init dans votre dépôt

   ```bash
   npx agent-note init --agent claude
   ```

   - Pour Codex CLI : `npx agent-note init --agent codex`
   - Pour Cursor : `npx agent-note init --agent cursor`
   - Pour Gemini CLI : `npx agent-note init --agent gemini`
   - Plusieurs agents à la fois : `npx agent-note init --agent claude cursor`

2. Committez les fichiers générés

   ```bash
   git add .claude/settings.json .github/workflows/agentnote-pr-report.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   - Codex CLI : versionner `.codex/config.toml` et `.codex/hooks.json`
   - Cursor : versionner `.cursor/hooks.json`
   - Gemini CLI : versionner `.gemini/settings.json`

3. Codez avec l'IA.
   Une fois les git hooks générés installés, le `git commit` normal suffit.
   `agent-note commit -m "..."` sert seulement de solution de secours quand les git hooks ne sont pas disponibles.
</Steps>

## Vérifiez votre configuration

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` affiche l'adaptateur actif, `capture:` résume ce que collectent les hooks actifs, `git:` indique si les git hooks gérés sont installés, et `commit:` indique si le `git commit` normal est la voie principale.
