Utilisez cette page quand vous connaissez déjà le flux Agent Note et que vous cherchez une commande ou un flag précis. Pour le premier setup, commencez par [Installation](./installation/).

## Tâches courantes

- Premier setup: `agent-note init`
- Voir ce qu'un commit a enregistré: `agent-note show`
- Comprendre pourquoi une ligne a changé: `agent-note why`
- Parcourir l'activité récente: `agent-note log`
- Générer un PR Report: `agent-note pr`
- Vérifier les hooks et l'adapter actif: `agent-note status`

## agent-note init

```bash
npx agent-note init --agent claude
```

| Flag | Description |
|---|---|
| `--agent <name...>` | Obligatoire. Un ou plusieurs adapters: `claude`, `codex`, `cursor`, `gemini` |
| `--dashboard` | Crée aussi le workflow Dashboard partagé |
| `--no-hooks` | Ignore la configuration des hooks agent |
| `--no-git-hooks` | Ignore l'installation des git hooks |
| `--no-action` | Ignore la création du workflow |
| `--no-notes` | Ignore la configuration auto-fetch des notes |
| `--hooks` | Installe seulement les hooks agent |
| `--action` | Crée seulement le workflow |

## agent-note show

```bash
npx agent-note show
npx agent-note show abc1234
```

Affiche les prompts, responses, fichiers touchés, agent, model et attribution pour un commit.

## agent-note why

Cherche avec `git blame` le Commit qui a modifié une ligne, puis affiche le contexte Agent Note lié à ce Commit.

```bash
npx agent-note why src/middleware/auth.ts:42
npx agent-note why src/middleware/auth.ts:42-55
npx agent-note why src/middleware/auth.ts:42:7
npx agent-note why src/middleware/auth.ts#L42
npx agent-note why src/middleware/auth.ts#L42-L55
npx agent-note why src/middleware/auth.ts#L42C7
npx agent-note why @src/middleware/auth.ts#L42
npx agent-note why https://github.com/owner/repo/blob/main/src/middleware/auth.ts#L42
npx agent-note why https://github.com/owner/repo/blob/main/src/middleware/auth.ts#L42-L55
npx agent-note why file:///workspace/repo/src/middleware/auth.ts#L42
npx agent-note why vscode://file/workspace/repo/src/middleware/auth.ts:42:7
```

<Aside>
  `why` accepte `path:line`, `path:line:column`, `path#Lline`, `path#Lline-Lend`, `path#LlineCcol`, les GitHub file URLs, `file://...`, `vscode://file...` et les paths préfixés par `@` copiés depuis une sortie d'AI Agent. Il évite le chemin manuel `git blame` → `git notes --ref=agentnote show <commit>`.
</Aside>

## agent-note log

```bash
npx agent-note log
npx agent-note log 20
```

Liste les commits récents avec leur ratio IA et le nombre de prompts enregistrés.

## agent-note pr

<Tabs>
  <TabItem label="Markdown">
  ```bash
  npx agent-note pr origin/main
  ```
  </TabItem>
  <TabItem label="Detail">
  ```bash
  npx agent-note pr origin/main --prompt-detail compact
  ```
  </TabItem>
  <TabItem label="JSON">
  ```bash
  npx agent-note pr origin/main --json
  ```
  </TabItem>
</Tabs>

<Aside>
  `compact` garde le PR Report ciblé sur les prompts qui expliquent le commit. `full` affiche tous les prompts sauvegardés.
</Aside>

## agent-note status

```bash
npx agent-note status
```

Vérifie l'adapter actif, ce que les hooks capturent, l'état des git hooks et le chemin de commit utilisé.

## agent-note deinit

```bash
npx agent-note deinit --agent claude
```

Ajoutez `--remove-workflow` pour supprimer aussi les workflows PR Report et Dashboard.

## agent-note commit

```bash
npx agent-note commit -m "feat: add auth"
```

<Aside type="tip">
  Inutile si les git hooks sont installés. Un `git commit` normal déclenche déjà le même enregistrement.
</Aside>
