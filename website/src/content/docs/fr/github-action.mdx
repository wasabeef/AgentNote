Cette page concerne le PR Report affiché dans les Pull Requests GitHub. Si vous voulez seulement terminer le setup local du CLI, commencez par [Installation](./installation/) ; revenez ici quand les reviewers doivent voir le résumé Agent Note.

## Mise en place rapide

Exécutez ceci une fois dans votre dépôt :

```bash
npx agent-note init --agent claude
```

Cela crée pour vous le workflow de rapport de PR :

```yaml
# .github/workflows/agentnote-pr-report.yml
name: Agent Note PR Report
on:
  pull_request:
    types: [opened, reopened, synchronize]
permissions:
  contents: read
  pull-requests: write
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0
      - uses: wasabeef/AgentNote@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

C’est tout. Chaque PR reçoit un rapport de session IA.

## Inputs

| Input | Valeur par défaut | Description |
|---|---|---|
| `base` | Branche de base de la PR | Branche utilisée pour la comparaison |
| `pr_output` | `description` | Destination du rapport de PR : `description`, `comment` ou `none` |
| `prompt_detail` | `compact` | Niveau de détail de l'historique des prompts dans le rapport de PR : `compact` garde le rapport ciblé, `full` affiche tous les prompts enregistrés |
| `dashboard` | `false` | Exécute le mode Dashboard au lieu du mode PR Report. Généralement généré par `agent-note init --dashboard` |

## Dashboard

```bash
npx agent-note init --agent claude --dashboard
```

<Aside type="tip">
  Cela ajoute aussi `.github/workflows/agentnote-dashboard.yml` pour configurer le Dashboard partagé.
</Aside>

## Outputs

| Output | Type | Description |
|---|---|---|
| `overall_ai_ratio` | number | Ratio IA sur l’ensemble de la PR (0-100) |
| `overall_method` | string | `line`, `file`, `mixed` ou `none` |
| `tracked_commits` | number | Nombre de commits avec des données agent-note |
| `total_commits` | number | Nombre total de commits dans la PR |
| `total_prompts` | number | Nombre total de prompts sur tous les commits |
| `json` | string | Rapport structuré complet |
| `markdown` | string | Rapport Markdown rendu |
| `should_deploy` | boolean string | Output du mode Dashboard qui indique au workflow appelant si Pages doit être publié |

## Ce que contient le rapport

Le rapport de PR contient :

- **Header** — `Total AI Ratio` avec une barre, puis `Model` en dessous
- **Reviewer Context** — zones modifiées, points de review et signaux d'intention pour les reviewers humains et les outils d'AI Review
- **Tableau** — ratio IA, nombre de prompts et fichiers par commit
- **Prompts** — entrées `📝 Context`, `🧑 Prompt` et `🤖 Response` par commit dans une section repliable

<Aside type="tip">
  Les hashes de commit du rapport pointent directement vers la page du commit sur GitHub.
</Aside>
