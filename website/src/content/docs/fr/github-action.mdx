## Mise en place rapide

Utilisez simplement le workflow créé par `agent-note init` :

```yaml
name: Agent Note
on:
  pull_request:
    types: [opened, synchronize]
permissions:
  contents: read
  pull-requests: write
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: wasabeef/AgentNote@v0
```

C’est suffisant. Chaque PR reçoit automatiquement un rapport de session IA.

## Inputs

| Input | Valeur par défaut | Description |
|---|---|---|
| `base` | Branche de base de la PR | Branche utilisée pour la comparaison |
| `pr_output` | `description` | Destination du rapport de PR : `description`, `comment` ou `none` |
| `dashboard` | `false` | Écrit aussi les fichiers du Dashboard pour la PR en cours |

### Utiliser la PR description et le Dashboard en même temps

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

<Aside type="tip">
  Activez `dashboard: true` uniquement si vous voulez aussi le Dashboard partagé. La page <a href="./dashboard/">Dashboard</a> explique comment s’articulent bundle, synchronisation Pages et déploiement.
</Aside>

## Outputs

| Output | Type | Description |
|---|---|---|
| `overall_ai_ratio` | number | Ratio IA sur l’ensemble de la PR (0-100) |
| `overall_method` | string | `line`, `file`, `mixed` ou `none` |
| `tracked_commits` | number | Nombre de commits avec des données agent-note |
| `total_commits` | number | Nombre total de commits dans la PR |
| `total_prompts` | number | Nombre total de prompts sur tous les commits |
| `json` | string | Rapport structuré complet |
| `markdown` | string | Rapport Markdown rendu |
| `dashboard_commits` | number | Nombre de tracked commits écrits dans le bundle du Dashboard |

## Ce que contient le rapport

Le rapport de PR contient :

- **Header** — `Total AI Ratio` avec une barre, puis `Model` en dessous
- **Tableau** — ratio IA, nombre de prompts et fichiers par commit
- **Prompts** — paires prompt / response par commit dans une section repliable

<Aside type="tip">
  Les hashes de commit du rapport pointent directement vers la page du commit sur GitHub.
</Aside>
