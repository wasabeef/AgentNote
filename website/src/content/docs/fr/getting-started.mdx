Agent Note est plus simple à adopter quand les pièces sont claires dès le départ. Les hooks collectent le contexte local, les git notes attachent ce contexte aux commits, et la GitHub Action transforme le tout en PR Report.

Cette page sert de carte. Les commandes d'installation sont sur la page suivante, pour que vous puissiez d'abord comprendre ce que vous activez.

## Ce que vous mettez en place

<CardGrid>
  <Card title="Capture locale" icon="laptop">
    Les hooks d'agent collectent prompts, réponses, fichiers touchés et signaux d'attribution pendant le travail.
  </Card>
  <Card title="Historique git-native" icon="document">
    Chaque commit reçoit une git note dans `refs/notes/agentnote`, sans modifier le contenu de la branche.
  </Card>
  <Card title="Contexte de Pull Request" icon="rocket">
    La GitHub Action lit ces notes et écrit le PR Report pour les reviewers.
  </Card>
  <Card title="Dashboard optionnel" icon="setting">
    Si l'équipe veut une vue partagée, Dashboard publie l'historique sur GitHub Pages.
  </Card>
</CardGrid>

## Parcours recommandé

<Steps>
1. Lisez ce modèle rapide.
2. Suivez [Installation](./installation/) pour exécuter `npx agent-note init`.
3. Ouvrez une Pull Request et vérifiez le PR Report.
4. Activez Dashboard seulement si vous voulez une page partagée pour l'équipe.
</Steps>
