Agent Note écrit le même format de git note pour tous les agents, mais chaque agent expose des hooks et transcripts différents. Cette page résume ce que vous pouvez attendre avant de l’activer pour une équipe.

## Matrice des capacités des agents

| Agent | Statut | Prompt | Response | Files | AI Ratio | Line Estimate |
|---|---|---|---|---|---|---|
| Claude Code | Full support | Oui | Oui | Oui | Oui | Par défaut |
| Codex CLI | Supported | Oui | Oui | Oui | Oui | Quand l'historique de patch Codex correspond au Commit final |
| Cursor | Supported | Oui | Oui | Oui | Oui | Quand les compteurs d'edit et le blob final correspondent |
| Gemini CLI | Preview | Oui | Oui | Oui | Oui | Pas encore |

<Aside>
  Files signifie qu'Agent Note peut montrer quels fichiers commités ont été touchés par l'Agent. Line Estimate signifie qu'il peut aussi estimer les lignes écrites par l'AI au lieu de seulement compter les fichiers. Preview signifie que l’implémentation suit le comportement public actuel, mais le format local de l’agent peut encore évoluer.
</Aside>

## Configuration recommandée

```bash
npx agent-note init --agent claude
```

Activez seulement les agents utilisés par votre équipe. Plusieurs agents peuvent cohabiter dans le même repository.

```bash
npx agent-note init --agent claude codex cursor gemini
```

Chaque developer doit lancer `init` une fois après un clone, car les git hooks vivent dans `.git/`.

## Choisir selon l'attente

| Besoin | Attendez-vous à |
|---|---|
| Attribution ligne par ligne la plus complète aujourd'hui | Claude Code |
| Support Codex CLI avec attribution prudente | Codex CLI |
| Historique d'éditeur Cursor et récupération des réponses | Cursor |
| Capture prompt / réponse avec Gemini CLI | Gemini CLI |

## Attribution

Line-level compte les lignes ajoutées par l’IA lorsque Agent Note peut comparer l’edit IA, les edits humains ultérieurs et le diff final. File-level est utilisé lorsque cette preuve n’est pas assez sûre.

Agent Note préfère toujours un fallback conservateur à une attribution devinée.

## Notes par agent

### Claude Code

Claude Code fournit les prompts, responses et edits via hooks. Agent Note enregistre l’état des fichiers avant et après l’edit IA, puis retire les edits humains arrivés ensuite.

### Codex CLI

Codex CLI est transcript-driven. Agent Note lit le transcript local et passe en line-level uniquement quand les comptes `apply_patch` correspondent au diff final.

### Cursor

Cursor utilise `afterFileEdit` / `afterTabFileEdit`, plus response hooks ou transcripts locaux. Le line-level est possible seulement si le blob final correspond encore au dernier edit IA.

### Gemini CLI

Gemini CLI utilise `BeforeTool` / `AfterTool` pour les fichiers et `BeforeAgent` / `AfterAgent` pour prompt / response. L’attribution reste file-level.

## Données manquantes

- Aucun note n’est écrit pour un commit sans fichier AI-authored.
- Un prompt ancien peut être ignoré s’il appartient à une autre fenêtre de commit.
- Une response peut être absente si l’agent ne l’expose pas.
- Certains generated artifacts et les chemins listés dans le `.agentnoteignore` à la racine du repository restent visibles dans Notes, PR Report et Dashboard; ils sont seulement exclus du dénominateur du ratio IA.
