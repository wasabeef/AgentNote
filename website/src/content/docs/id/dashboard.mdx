## Yang akan Anda dapatkan

Dashboard memberi tim Anda satu URL bersama untuk melihat riwayat Agent Note.

- Riwayat PR terbuka dan merged di satu tempat
- Commit, prompt, dan attribution tanpa membuka setiap PR satu per satu
- Melihat context khusus tampilan sebelum prompt pendek saat Agent Note memulihkan reference atau scope
- Satu URL GitHub Pages bersama, biasanya di bawah `/dashboard/`

Gunakan ini ketika laporan PR saja belum cukup dan tim membutuhkan tampilan bersama.

## Sebelum mulai

Anda hanya perlu Dashboard jika memang ingin tampilan web bersama.

- Jika laporan sesi AI di dalam pull request sudah cukup, berhenti di [GitHub Action](./github-action/).
- Jika ingin tampilan Pages bersama untuk riwayat terbuka dan merged, lanjutkan di sini.

## Quick setup

1. Jalankan setup Dashboard sekali.

```bash
npx agent-note init --agent claude --dashboard
```

2. Commit kedua file workflow.

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. Di GitHub Pages, pilih **GitHub Actions** sebagai source.

Workflow Dashboard yang dihasilkan memakai `wasabeef/AgentNote@v0` dengan `dashboard: true` untuk restore, sync, build, upload artifact, dan persistensi note, lalu memublikasikan tampilan bersama di `/dashboard/`. Jika workflow yang sama sudah mengunggah artifact GitHub Pages, Agent Note menambahkan Dashboard ke direktori `dashboard/` di artifact itu, bukan mengganti situs yang sudah ada. Jika workflow lain sudah memublikasikan Pages, Agent Note melewati publikasi standalone; letakkan step Dashboard sebelum upload artifact di workflow tersebut untuk menggabungkannya.

## URL publik

Biasanya Dashboard dipublikasikan di sini:

- `https://<owner>.github.io/<repo>/dashboard/`

Contoh:

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Jika Anda memublikasikan dengan GitHub Pages, workflow Pages akan mengirim nilai `SITE` dan `BASE` yang spesifik untuk repositori saat build. Pada kebanyakan repositori, workflow contoh bisa dipakai tanpa mengubah Dashboard source.
</Aside>

## Deploy pertama

Bahkan repositori baru bisa mulai mengumpulkan data Dashboard sebelum deploy produksi pertama. Jika Pull Request Deploy diizinkan, URL publik Pages bisa muncul setelah run `pull_request` pertama yang sukses. Jika tidak, URL itu muncul setelah deploy pertama yang sukses dari `default branch`.

## Cara kerjanya

- `refs/notes/agentnote` tetap menjadi Source of Truth
- `wasabeef/AgentNote@v0` dalam mode Dashboard menulis snapshot note ke `gh-pages/dashboard/notes/*.json`
- GitHub Pages menyajikan Dashboard yang sudah dibangun di `/dashboard/`

Repositori tidak meng-commit contoh JSON Dashboard. Dashboard baru tetap kosong sampai workflow memulihkan atau membuat note file. `pull_request` memublikasikan status Open, lalu push ke `default branch` menggantinya dengan status Merged.
