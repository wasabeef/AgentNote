## Yang akan Anda dapatkan

Dashboard memberi tim Anda satu URL bersama untuk melihat riwayat Agent Note.

- Riwayat PR terbuka dan merged di satu tempat
- Commit, prompt, dan attribution tanpa membuka setiap PR satu per satu
- Satu URL GitHub Pages bersama, biasanya di bawah `/dashboard/`

Gunakan ini ketika PR report saja belum cukup dan tim membutuhkan tampilan bersama.

## Sebelum mulai

Anda hanya perlu Dashboard jika memang ingin tampilan web bersama.

- Jika laporan sesi AI di dalam pull request sudah cukup, berhenti di [GitHub Action](./github-action/).
- Jika ingin tampilan Pages bersama untuk riwayat terbuka dan merged, lanjutkan di sini.

## Quick setup

Pertama, biarkan Action menulis file Dashboard untuk setiap PR:

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Ini akan membuat atau memperbarui `packages/dashboard/public/notes/*.json` untuk PR saat ini. Setelah itu workflow Pages Anda menangani restore, persist, dan publish file tersebut.

## URL publik

Biasanya Dashboard dipublikasikan di sini:

- `https://<owner>.github.io/<repo>/dashboard/`

Contoh:

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Jika Anda memublikasikan dengan GitHub Pages, workflow Pages akan mengirim nilai `SITE` dan `BASE` yang spesifik untuk repo saat build. Pada kebanyakan repo, workflow contoh bisa dipakai tanpa mengubah `packages/dashboard`.
</Aside>

## Deploy pertama

Bahkan repo baru bisa mulai mengumpulkan data Dashboard sebelum deploy produksi pertama. URL publik Pages baru muncul setelah deploy pertama ke `main`.

## Cara kerjanya

- `packages/dashboard/` adalah aplikasi Astro untuk tampilan Dashboard
- `packages/dashboard/public/notes/*.json` adalah bundle yang dibaca saat build
- `gh-pages/dashboard/notes/*.json` adalah penyimpanan permanen untuk riwayat bersama

Repo ini tidak menyertakan JSON contoh untuk Dashboard. Dashboard baru akan tetap kosong sampai workflow Anda me-restore atau menghasilkan note files. Banyak tim memperbarui PR notes di `pull_request`, lalu memublikasikan status merged dari `main`.
