## Yang akan Anda dapatkan

Dashboard memberi tim Anda satu URL bersama untuk melihat riwayat Agent Note.

- Riwayat PR terbuka dan merged di satu tempat
- Commit, prompt, dan attribution tanpa membuka setiap PR satu per satu
- Satu URL GitHub Pages bersama, biasanya di bawah `/dashboard/`

Gunakan ini ketika PR report saja belum cukup dan tim membutuhkan tampilan bersama.

## Sebelum mulai

Anda hanya perlu Dashboard jika memang ingin tampilan web bersama.

- Jika laporan sesi AI di dalam pull request sudah cukup, berhenti di [GitHub Action](./github-action/).
- Jika ingin tampilan Pages bersama untuk riwayat terbuka dan merged, lanjutkan di sini.

## Quick setup

1. Run Dashboard setup once.

```bash
npx agent-note init --agent claude --dashboard
```

2. Commit both workflow files.

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. In GitHub Pages, choose **GitHub Actions** as the source.

The generated Dashboard workflow restores data from `gh-pages`, refreshes it from `refs/notes/agentnote`, and publishes the shared `/dashboard/` view.

## URL publik

Biasanya Dashboard dipublikasikan di sini:

- `https://<owner>.github.io/<repo>/dashboard/`

Contoh:

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Jika Anda memublikasikan dengan GitHub Pages, workflow Pages akan mengirim nilai `SITE` dan `BASE` yang spesifik untuk repo saat build. Pada kebanyakan repo, workflow contoh bisa dipakai tanpa mengubah Dashboard source.
</Aside>

## Deploy pertama

Bahkan repo baru bisa mulai mengumpulkan data Dashboard sebelum deploy produksi pertama. URL publik Pages baru muncul setelah deploy pertama ke `default branch`.

## Cara kerjanya

- `refs/notes/agentnote` remains the source of truth
- The generated Dashboard workflow writes shared snapshots to `gh-pages/dashboard/notes/*.json`
- GitHub Pages serves the built Dashboard at `/dashboard/`

The repository does not commit sample Dashboard JSON. A fresh Dashboard stays empty until the workflow restores or generates note files. `pull_request` refreshes open history, and `default branch` publishes the merged state.
