## Yang Anda Dapatkan

Dashboard memberi tim Anda satu URL bersama untuk riwayat Agent Note.

- Melihat riwayat PR yang open dan merged di satu tempat
- Meninjau commit, prompt, dan attribution tanpa membuka setiap PR
- Membagikan satu URL GitHub Pages, biasanya di bawah `/dashboard/`

Gunakan ini ketika PR report saja tidak lagi cukup dan Anda membutuhkan tampilan bersama untuk tim.

## Sebelum Mulai

Anda hanya perlu dashboard jika ingin tampilan web bersama.

- Jika AI session report di pull request sudah cukup, berhenti di [GitHub Action](./github-action/).
- Jika ingin tampilan Pages bersama untuk riwayat open dan merged, lanjutkan di sini.

## Quick Setup

Pertama, biarkan Action menulis dashboard bundle untuk setiap PR:

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
    dashboard_url: https://docs.example.com/dashboard/
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Ini akan membuat atau memperbarui `packages/dashboard/public/notes/*.json` untuk PR saat ini. Setelah itu Pages workflow Anda bisa me-restore, menyimpan, dan mem-publish file tersebut.

<Aside type="tip">
  `dashboard_url` bersifat optional. Setel hanya setelah dashboard benar-benar public, agar PR report tidak mengarah ke dead link.
</Aside>

## URL publik

Biasanya dashboard dipublikasikan di sini:

- `https://<owner>.github.io/<repo>/dashboard/`

Contoh:

- `https://wasabeef.github.io/AgentNote/dashboard/`

## Konfigurasi deploy

`packages/dashboard` tetap memakai default lokal `/`, jadi Pages workflow Anda harus menyuntikkan nilai khusus repo saat build:

- `SITE=https://<owner>.github.io`
- `BASE=/<repo>/dashboard`
- `PUBLIC_REPO=<owner>/<repo>`
- `PUBLIC_REPO_URL=https://github.com/<owner>/<repo>` (optional)

Dengan begitu package tetap generic, tetapi URL publik untuk tiap repository tetap benar.

## Alur yang disarankan

1. Aktifkan `dashboard: true` di GitHub Action.
2. Restore `gh-pages/dashboard/notes` ke `packages/dashboard/public/notes`.
3. Pada `pull_request` (`opened`, `reopened`, `synchronize`), tulis ulang note files untuk PR tersebut lalu simpan kembali ke `gh-pages`.
4. Pada `push` ke `main`, tandai PR notes yang cocok sebagai `merged`, build ulang site, lalu deploy shared Pages URL.

Ini menjaga `main` tetap bersih dari JSON generated, sambil tetap memungkinkan satu dashboard bersama menampilkan riwayat PR open dan merged.

## Deploy pertama

Repo yang benar-benar baru tetap bisa mengumpulkan dashboard data sebelum deploy production pertama, tetapi URL public Pages baru muncul setelah deploy pertama dari `main`.

## Cara kerjanya

- `packages/dashboard/` adalah Astro app interaktif
- `packages/dashboard/public/notes/*.json` adalah working bundle yang dipakai saat build
- `gh-pages/dashboard/notes/*.json` adalah durable store untuk riwayat bersama

Repository ini tidak menyertakan sample dashboard JSON. Dashboard baru akan tetap kosong sampai workflow Anda me-restore atau membuat note files.
