## Penyimpanan

<CardGrid>
  <Card title="Lapisan 1: data sementara lokal" icon="laptop">
    Data sesi di `.git/agentnote/sessions/`. File JSONL untuk prompt, perubahan, dan blob hash. Dirotasi setelah setiap commit. Tidak pernah dipush.
  </Card>
  <Card title="Lapisan 2: Git Notes" icon="document">
    Catatan permanen di `refs/notes/agentnote`. Satu JSON per commit. Bisa dipush, difetch, dan dibagikan ke tim.
  </Card>
</CardGrid>

### File sesi

| File | Tujuan |
|---|---|
| `prompts.jsonl` | Satu prompt per baris dengan nomor turn |
| `changes.jsonl` | File yang disentuh AI dengan post-edit blob hash |
| `pre_blobs.jsonl` | Keadaan file sebelum setiap edit AI |
| `heartbeat` | Stempel waktu aktivitas sesi |
| `turn` | Penghitung monoton yang bertambah di setiap prompt |

### Note Schema

```json
{
  "v": 1,
  "agent": "claude",
  "session_id": "a1b2c3d4-...",
  "model": "claude-sonnet-4-20250514",
  "interactions": [
    {
      "prompt": "Implementasikan middleware auth JWT",
      "response": "Saya akan membuat middleware-nya...",
      "files_touched": ["src/auth.ts"],
      "tools": ["Edit"]
    }
  ],
  "files": [
    { "path": "src/auth.ts", "by_ai": true },
    { "path": "CHANGELOG.md", "by_ai": false }
  ],
  "attribution": {
    "ai_ratio": 73,
    "method": "line",
    "lines": { "ai_added": 146, "total_added": 200, "deleted": 12 }
  }
}
```

<Aside type="tip">
  Daftar prompt berasal dari percakapan kausal di sekitar final diff, bukan dari seluruh backlog sesi. Dalam mode file-level, Agent Note hanya menyimpan jalur edit terakhir dan prompt-only context yang mengarah ke sana.
</Aside>

<Aside>
  Generated artifact umum dikeluarkan dari denominator `AI ratio` secara best-effort, termasuk output build web, Flutter registrant, Dart codegen, Go protobuf, Rust binding, dan generated source Swift / Kotlin.
</Aside>

## Siklus hidup sesi

| Event | Yang terjadi |
|---|---|
| **SessionStart** | Membuat direktori sesi dan menulis heartbeat |
| **UserPromptSubmit** | Menambahkan prompt, menaikkan counter turn, dan merotasi log |
| **PreToolUse Edit/Write** | Menangkap pre-edit blob hash secara sinkron |
| **PostToolUse Edit/Write** | Menangkap post-edit blob hash dan path file |
| **Stop** | Mencatat event stop (heartbeat tetap aktif — Stop = akhir response, bukan akhir sesi) |

<Aside>
  Git hook memverifikasi kesegaran sesi melalui heartbeat sebelum menyisipkan trailer atau merekam notes. Heartbeat diperbarui pada SessionStart dan setiap prompt.
</Aside>

<Aside type="tip">
  Agent Note memahami agent yang sedang aktif. Metadata sesi menyimpan adapter aktif, sehingga `show`, `status`, dan perekaman commit nanti bisa menyelesaikan format transcript yang benar.
</Aside>

## Git Notes

Git notes tidak terlihat di daftar branch, UI GitHub, atau CI — tetapi bisa dipush dan difetch.

```bash
# Bagikan ke tim
git push origin refs/notes/agentnote

# Ambil dari remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# Lihat note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` menyiapkan auto-fetch, dan hook `pre-push` menangani auto-push. Tidak perlu perintah manual.
</Aside>
