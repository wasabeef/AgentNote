### File sesi

| File | Tujuan |
|---|---|
| `prompts.jsonl` | Satu prompt per baris dengan nomor turn |
| `changes.jsonl` | File yang disentuh AI dengan post-edit blob hash |
| `pre_blobs.jsonl` | Keadaan file sebelum setiap edit AI |
| `heartbeat` | Stempel waktu aktivitas sesi |
| `turn` | Penghitung monoton yang bertambah di setiap prompt |

## Cara mengumpulkan `📝 Context`

`📝 Context` adalah catatan khusus tampilan untuk prompt pendek seperti "continue" atau "kerjakan yang berikutnya". Context membantu pembaca memahami prompt tanpa mengubah attribution.

<img src="/AgentNote/images/context-dashboard-example.png" alt="Agent Note Dashboard showing Context before a short prompt" style="width: 100%; border-radius: 20px; border: 1px solid rgba(148, 163, 184, 0.18); box-shadow: 0 24px 80px rgba(15, 23, 42, 0.28);" />

- `reference` context diambil dari response tepat sebelumnya ketika response itu menyebut path file yang berubah, nama file, atau code identifier dari diff final.
- `scope` context diambil dari response saat ini ketika awal response menjelaskan pekerjaan yang sedang dilakukan.
- Agent Note tidak memakai kata persetujuan singkat seperti `yes`, `continue`, atau `お願いします` sebagai aturan seleksi. Jika tidak ada petunjuk konkret tentang file atau code, Context dilewati.
- Context tidak pernah mengubah `files_touched`, prompt ownership, AI ratio, attribution method, atau apakah git note disimpan.

## Agent Note vs Entire

Agent Note dan Entire menyelesaikan masalah yang berdekatan, tetapi berbeda.

- Agent Note menghubungkan git commit dengan percakapan AI yang membuatnya. Entire adalah checkpoint product yang lebih luas dengan workflow rewind / resume.
- Agent Note menyimpan record permanen di `refs/notes/agentnote`; file sementara sesi tetap berada di `.git/agentnote/`. Entire berpusat pada checkpoint metadata dan hosted web application.
- Dashboard Agent Note adalah static GitHub Pages opsional yang didukung git notes. Agent Note tetap bisa dipakai tanpa hosted service.
- Agent Note tidak mencoba memulihkan workspace state lama. Fokusnya adalah reviewability: mengapa commit berubah, prompt mana yang terlibat, dan seberapa besar diff yang AI-authored.

<Aside type="tip">
  Daftar prompt berasal dari jendela percakapan antar-commit, bukan dari seluruh backlog sesi. Agent Note mempertahankan prompt planning, clarification, dan review yang berguna untuk commit saat ini, sambil memangkas riwayat prompt yang dikutip dan edit turn yang tidak bertahan di commit ini.
</Aside>

<Aside>
  Generated artifact umum dikeluarkan dari denominator `AI ratio` secara best-effort memakai sinyal spesifik seperti path framework/cache, nama atau suffix generated, dan header file yang di-commit. Nama direktori generik seperti `build/`, `dist/`, `gen/`, `generated/`, dan `target/` saja tidak cukup. Jika repository Anda meng-commit bundle atau generated output yang harus tetap terlihat tetapi tidak memengaruhi AI Ratio, tulis polanya di `.agentnoteignore` di repository root.
</Aside>

## Siklus hidup sesi

| Event | Yang terjadi |
|---|---|
| **SessionStart** | Membuat direktori sesi dan menulis heartbeat |
| **UserPromptSubmit** | Menambahkan prompt, menaikkan counter turn, dan merotasi log |
| **PreToolUse Edit/Write** | Menangkap pre-edit blob hash secara sinkron |
| **PostToolUse Edit/Write** | Menangkap post-edit blob hash dan path file |
| **Stop** | Mencatat event stop (heartbeat tetap aktif — Stop = akhir response, bukan akhir sesi) |

<Aside>
  Git hook memverifikasi kesegaran sesi melalui heartbeat dan memastikan ada data sesi yang dapat direkam sebelum menyisipkan trailer atau merekam notes. Heartbeat diperbarui oleh hook event selama turn yang panjang; sesi metadata-only dilewati. Path transcript saja adalah metadata pendukung, bukan data yang dapat direkam.
</Aside>

<Aside type="tip">
  Agent Note memahami agent yang sedang aktif. Metadata sesi menyimpan adapter aktif, sehingga `show`, `status`, dan perekaman commit nanti bisa menyelesaikan format transcript yang benar.
</Aside>

## Git Notes

Git notes tidak terlihat di daftar branch, UI GitHub, atau CI — tetapi bisa dipush dan difetch.

```bash
# Bagikan ke tim
git push origin refs/notes/agentnote

# Ambil dari remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# Lihat note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` menyiapkan auto-fetch, dan hook `pre-push` menangani auto-push. Tidak perlu perintah manual.
</Aside>
