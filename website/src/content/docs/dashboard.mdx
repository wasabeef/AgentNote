## What You Get

The Dashboard gives your team one shared URL for Agent Note history.

- Browse Open / Merged PR history in one place
- Inspect Commits, Prompts, and Attribution without opening each PR
- See display-only Context before short prompts when Agent Note can recover reference or scope
- Share one GitHub Pages URL with your team, usually `/dashboard/`

Use it when the PR Report is not enough and you want one team-facing view of recent work.

## Before You Start

You only need the Dashboard if you want one shared view for the team.

- If you only want AI session reports inside pull requests, stop at [GitHub Action](./github-action/).
- If you want a shared Pages view for Open / Merged history, continue here.

## Quick Setup

1. Run Dashboard setup once:

```bash
npx agent-note init --agent claude --dashboard
```

2. Commit both workflow files:

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. In GitHub, enable Pages for the repository and choose **GitHub Actions** as the source.

The generated Dashboard workflow uses `wasabeef/AgentNote@v0` with `dashboard: true` to restore, sync, build, upload the artifact, and persist notes, then publishes the shared `/dashboard/` view for you. If the same job already uploads a GitHub Pages artifact, Agent Note adds the Dashboard to that artifact's `dashboard/` directory instead of replacing the existing site. If another job or workflow already publishes Pages, Agent Note skips standalone publishing; place Agent Note in the same job before `actions/upload-pages-artifact` to combine a docs site and Dashboard.

## Public URL

The Dashboard is typically published here:

- `https://<owner>.github.io/<repo>/dashboard/`

For example:

- `https://wasabeef.github.io/AgentNote/dashboard/`

## PR Previews

If your repository's `github-pages` environment allows Pull Request Deploys, the shared Dashboard can show the Open PR state before merge.

- `pull_request` runs publish the current Open state
- `default branch` pushes replace it with the Merged state
- if environment protection blocks Pull Request Deploys, the shared Dashboard updates after merge instead

This is controlled by your repository's GitHub Pages environment settings, not by Agent Note itself.

## First Deploy

A brand-new Repository can accumulate Dashboard note data before the Dashboard is published for the first time. If Pull Request Deploys are allowed, the shared Pages URL can appear after the first successful `pull_request` run. Otherwise it appears after the first successful deploy from `default branch`.

## How It Works

- `refs/notes/agentnote` remains the source of truth
- `wasabeef/AgentNote@v0` in Dashboard mode writes note snapshots to `gh-pages/dashboard/notes/*.json`
- GitHub Pages serves the built Dashboard at `/dashboard/`

The Repository does not commit sample JSON for the Dashboard. A new Dashboard stays empty until the workflow restores or generates note files. Pull Request runs publish the Open state, and `default branch` pushes replace it with the Merged state.
