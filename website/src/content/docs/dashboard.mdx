## What You Get

The Dashboard gives your team one shared URL for Agent Note history.

- Browse open and merged PR history in one place
- Inspect commits, prompts, and attribution without opening each PR
- Share one GitHub Pages URL with your team, usually `/dashboard/`

Use it when the PR report is not enough and you want a team-facing view of recent work.

## Before You Start

You only need the Dashboard if you want a shared web view.

- If you only want AI session reports inside pull requests, stop at [GitHub Action](./github-action/).
- If you want a shared Pages view for open and merged history, continue here.

## Quick Setup

1. Run Dashboard setup once:

```bash
npx agent-note init --agent claude --dashboard
```

2. Commit both workflow files:

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. In GitHub, enable Pages for the repository and choose **GitHub Actions** as the source.

The generated Dashboard workflow restores note data from `gh-pages`, refreshes it from `refs/notes/agentnote`, and publishes the shared `/dashboard/` view for you.

## Public URL

The Dashboard is typically published here:

- `https://<owner>.github.io/<repo>/dashboard/`

For example:

- `https://wasabeef.github.io/AgentNote/dashboard/`

## First Deploy

A brand-new repo can accumulate Dashboard note data before the first public publish, and the shared Pages URL can appear as soon as the first successful Dashboard workflow run completes.

## How It Works

- `refs/notes/agentnote` remains the source of truth
- The generated Dashboard workflow writes shared note snapshots to `gh-pages/dashboard/notes/*.json`
- GitHub Pages serves the built Dashboard at `/dashboard/`

The repository does not commit sample Dashboard JSON. A fresh Dashboard stays empty until the workflow restores or generates note files. Pull requests publish the open state, and `default branch` pushes replace it with the merged state.
