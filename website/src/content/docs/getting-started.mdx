## Installation

<Steps>
1. **Run init** in your repository

   ```bash
   npx agent-note init --agent claude
   ```

   For Codex CLI:

   ```bash
   npx agent-note init --agent codex
   ```

   For Cursor:

   ```bash
   npx agent-note init --agent cursor
   ```

   For Gemini CLI:

   ```bash
   npx agent-note init --agent gemini
   ```

   Multiple agents at once:

   ```bash
   npx agent-note init --agent claude --agent cursor
   ```

   This configures:
   - Agent hooks (`.claude/settings.json` for Claude Code, `.codex/config.toml` + `.codex/hooks.json` for Codex CLI, `.cursor/hooks.json` for Cursor, `.gemini/settings.json` for Gemini CLI)
   - Git hooks (`prepare-commit-msg`, `post-commit`, `pre-push`)
   - GitHub Action workflow (`.github/workflows/agentnote.yml`)
   - Auto-fetch for git notes on pull

2. **Commit** the generated files

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   Codex repositories commit `.codex/config.toml`, `.codex/hooks.json`, and `.github/workflows/agentnote.yml`.
   Cursor repositories commit `.cursor/hooks.json` and `.github/workflows/agentnote.yml`.
   Gemini CLI repositories commit `.gemini/settings.json` and `.github/workflows/agentnote.yml`.

3. **Done.** Start coding with your agent.
   Claude Code and Codex can use normal `git commit`.
   Cursor preview also works with normal `git commit` when the generated git hooks are installed.
   Use `agent-note commit -m "..."` as a fallback when git hooks are unavailable.
</Steps>

## New Clone Setup

Each developer runs init after cloning to install local git hooks:

```bash
git clone <repo> && cd <repo>
npx agent-note init --agent claude
```

<Aside>
  Git hooks are local to `.git/` and not committed. Every developer must run `init` once per clone.
</Aside>

## What Happens Automatically

After init, plain `git commit` will:

1. **Inject** an `Agentnote-Session` trailer into the commit message
2. **Record** prompts, responses, and file attribution as a git note
3. **Push** notes to the remote on `git push`

Cursor preview also restores prompt / response pairs from response hooks or local transcripts.
If git hooks are unavailable, `agent-note commit` remains the fallback wrapper.

## Init Flags

| Flag | Description |
|---|---|
| `--no-hooks` | Skip agent hook configuration |
| `--no-git-hooks` | Skip git hook installation |
| `--no-action` | Skip workflow creation |
| `--no-notes` | Skip auto-fetch config |
| `--hooks` | Only install agent hooks |
| `--action` | Only create workflow |
| `--agent <name>` | Choose the agent adapter, for example `claude`, `codex`, `cursor`, or `gemini` |

## Uninstall

```bash
npx agent-note deinit --agent claude
```

This removes agent hooks and git hooks. Add `--remove-workflow` to also delete the GitHub Action workflow.
