## What it does

<CardGrid stagger>
  <Card title="Keep AI-authored lines on record" icon="pencil">
    Finer than `git blame`. Claude Code is full line-level; other agents upgrade when conditions match.
  </Card>
  <Card title="Prompts and responses, saved alongside" icon="document">
    Model name and tool usage too — all bundled into git notes with the change.
  </Card>
  <Card title="Automatic PR reports" icon="rocket">
    AI ratio, prompt count, commit hashes. Reviewers see context before code.
  </Card>
  <Card title="Written only to git notes" icon="setting">
    Branches stay clean. Push / fetch to share with the team.
  </Card>
</CardGrid>

## Setup

<Steps>
1. Run init in your repository

   ```bash
   npx agent-note init --agent claude
   ```

   - Codex CLI: `npx agent-note init --agent codex`
   - Cursor: `npx agent-note init --agent cursor`
   - Gemini CLI: `npx agent-note init --agent gemini`
   - Multiple agents at once: `npx agent-note init --agent claude --agent cursor`

2. Commit the generated files

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   - Codex CLI: commit `.codex/config.toml` and `.codex/hooks.json`
   - Cursor: commit `.cursor/hooks.json`
   - Gemini CLI: commit `.gemini/settings.json`

3. Code with AI.
   Plain `git commit` works when the generated git hooks are installed.
   `agent-note commit -m "..."` remains the fallback wrapper if git hooks are unavailable.
</Steps>

## Check Your Setup

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` shows the enabled adapter, `capture:` summarizes what the active agent hooks collect, `git:` shows whether the managed repo-local git hooks are installed, and `commit:` shows whether normal `git commit` is the primary path or whether fallback mode is active.
