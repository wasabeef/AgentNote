## What Agent Note Does

<CardGrid stagger>
  <Card title="Line-Level AI Attribution" icon="pencil">
    Claude Code gets full line-level attribution. Codex upgrades to line-level when transcript patch counts match the final commit.
  </Card>
  <Card title="Session Tracking" icon="document">
    Prompts, responses, model name, and tool usage — all linked to the commits they produced.
  </Card>
  <Card title="PR Reports" icon="rocket">
    Automatic AI session reports on pull requests with per-commit ratio, line counts, and linked hashes.
  </Card>
  <Card title="Git Native" icon="setting">
    Stored as git notes — invisible to branches and CI. Pushable, fetchable, shareable.
  </Card>
</CardGrid>

## Setup in 60 Seconds

<Steps>
1. Run init in your repository

   ```bash
   npx @wasabeef/agentnote init
   ```

2. Commit the generated files

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agentnote"
   git push
   ```

   For Codex CLI, run `npx @wasabeef/agentnote init --agent codex` and commit `.codex/config.toml` plus `.codex/hooks.json` instead of `.claude/settings.json`.
   For Cursor, run `npx @wasabeef/agentnote init --agent cursor` and commit `.cursor/hooks.json`.
   For Gemini CLI, run `npx @wasabeef/agentnote init --agent gemini` and commit `.gemini/settings.json`.

3. Code with AI.
   Plain `git commit` works when the generated git hooks are installed.
   `agentnote commit -m "..."` remains the fallback wrapper if git hooks are unavailable.
</Steps>

## Check Your Setup

```bash
agentnote status
```

```text
agentnote v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` shows the enabled adapter, `capture:` summarizes what the active agent hooks collect, `git:` shows whether the managed repo-local git hooks are installed, and `commit:` shows whether normal `git commit` is the primary path or whether fallback mode is active.

## Support Matrix

| Agent | Status | Notes |
|---|---|---|
| Claude Code | Full support | Line-level attribution via hook-captured blob pairs |
| Codex CLI | Preview | Transcript-driven attribution, with line-level upgrade when patch counts match the commit |
| Cursor | Preview | `afterFileEdit` / `afterTabFileEdit`-driven attribution, with safe line-level upgrade when the committed blob still matches the AI edit |
| Gemini CLI | Preview | `BeforeTool`/`AfterTool`-driven file attribution via `.gemini/settings.json` hooks |

## Capability Snapshot

| Capability | Claude Code | Codex CLI | Cursor | Gemini CLI |
|---|---|---|---|---|
| Plain `git commit` with generated git hooks | Yes | Yes | Yes | Yes |
| `agentnote commit` fallback | Yes | Yes | Yes | Yes |
| Prompt / response recovery | Hook-native | Local transcript | Response hooks or local transcripts | `BeforeAgent`/`AfterAgent` hooks |
| Default attribution | Line-level | File-level | File-level | File-level |
| Safe line-level upgrade | Default path | When transcript patch counts match the commit | When `afterFileEdit` / `afterTabFileEdit` counts match and the committed blob still matches the last AI edit | Not yet available |
