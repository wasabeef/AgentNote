## できること

Dashboard は、Agent Note の履歴をチームで共有するための 1 つの URL です。

- open / merged 両方の PR 履歴を 1 か所で見られる
- commit、prompt、attribution をまとめてたどれる
- GitHub Pages の `/dashboard/` 配下に 1 つの共有 URL として公開できる

PR Description だけでは足りず、チーム向けの一覧画面がほしいときに使います。

## 先に決めること

Dashboard は web の共有画面が必要なときだけ使います。

- PR の中に AI レポートが出れば十分なら、[GitHub Action](./github-action/) だけで OK です。
- open / merged 履歴をまとめて見たいなら、このページの設定を足します。

## クイックセットアップ

1. まずは 1 回だけ次を実行します。

```bash
npx agent-note init --agent claude --dashboard
```

2. 生成された 2 本の workflow をコミットします。

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. GitHub の Pages 設定で、公開元に **GitHub Actions** を選びます。

生成された Dashboard workflow が `gh-pages` から note data を復元し、`refs/notes/agentnote` から更新して、共有用の `/dashboard/` を公開します。

## 公開 URL

通常は次の URL に公開します。

- `https://<owner>.github.io/<repo>/dashboard/`

たとえば:

- `https://wasabeef.github.io/AgentNote/dashboard/`

## 最初の deploy

brand-new repo でも最初の production deploy 前から Dashboard data 自体は蓄積できますが、public な Pages URL ができるのは最初の `default branch` deploy 後です。

## 仕組み

- `refs/notes/agentnote` が source of truth のまま残ります
- 生成された Dashboard workflow が `gh-pages/dashboard/notes/*.json` を共有用の保存先として更新します
- GitHub Pages が `/dashboard/` で build 済み Dashboard を配信します

repo 自体には sample Dashboard JSON をコミットしません。fresh な Dashboard は workflow が note file を復元または生成するまで空です。`pull_request` で open な履歴を更新し、`default branch` で merged 状態を公開します。
