```json
  }

```

## `📝 Context` の集め方

`📝 Context` は、`続けて` や `次の作業にうつって` のような短い prompt を読みやすくするための display-only metadata です。attribution には影響しません。

- `reference` context は直前の response から取ります。最終 diff に含まれる changed file path、file basename、code identifier などの強い structural anchor がある場合だけ使います。
- `scope` context は現在の response から取ります。response の冒頭が structural anchor を含んだ作業範囲を説明している場合だけ使います。
- `はい`、`continue`、`お願いします` のような approval keyword は選択条件にしません。structural anchor がなければ Context は付けません。
- Context は `files_touched`、prompt ownership、AI ratio、attribution method、git note を保存するかどうかを変えません。

## Agent Note と Entire の違い

Agent Note と Entire は近い領域ですが、目指している workflow が違います。

- Agent Note は git commit と、その commit を生んだ AI conversation を紐づけます。Entire は rewind / resume を含む checkpoint product です。
- Agent Note の永続データは `refs/notes/agentnote` に保存され、一時データは `.git/agentnote/` に置かれます。Entire は checkpoint metadata と hosted web application を中心にします。
- Agent Note の Dashboard は git notes を元にした optional static GitHub Pages です。hosted service なしでも使えます。
- Agent Note は古い workspace state を復元する tool ではありません。commit がなぜ変わったか、どの prompt が関わったか、diff のどれくらいが AI-authored かを review しやすくする tool です。

<Aside type="tip">
  Prompt 一覧はセッション全体ではなく、commit から次の commit までの会話 window から選ばれます。現在の commit に必要な planning、clarification、review の prompt は残しつつ、引用された過去の prompt 履歴や、この commit に残らなかった edit turn は削ります。
</Aside>

<Aside>
  代表的な generated artifact は、framework / cache path、generated filename / suffix、committed file header などの具体的な signal で best-effort に `AI ratio` の分母から除外されます。`build/`、`dist/`、`gen/`、`generated/`、`target/` のような汎用ディレクトリ名だけでは generated 扱いしません。
</Aside>
