## Agent Note でできること

<CardGrid stagger>
  <Card title="行単位の AI 著作判定" icon="pencil">
    Claude Code は 3-diff で完全な行単位判定。Codex は transcript の patch 行数と commit が一致した場合に行単位へ昇格します。
  </Card>
  <Card title="セッション記録" icon="document">
    プロンプト・レスポンス・使用モデル・ツール情報をコミットと一緒に記録します。
  </Card>
  <Card title="PR レポート" icon="rocket">
    プルリクエストに AI セッションレポートを自動投稿。コミットごとの AI 比率・変更行数・リンク付きハッシュを表示。
  </Card>
  <Card title="Git ネイティブ" icon="setting">
    git notes で保存するため、ブランチや CI に影響しません。push・fetch でチーム共有も簡単。
  </Card>
</CardGrid>

## 60 秒でセットアップ

<Steps>
1. リポジトリで init を実行

   ```bash
   npx @wasabeef/agentnote init
   ```

   Codex CLI の場合:

   ```bash
   npx @wasabeef/agentnote init --agent codex
   ```

   Cursor の場合:

   ```bash
   npx @wasabeef/agentnote init --agent cursor
   ```

2. 生成されたファイルをコミット

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agentnote"
   git push
   ```

   Codex CLI の場合は `.claude/settings.json` の代わりに `.codex/config.toml` と `.codex/hooks.json` をコミットします。
   Cursor の場合は `.cursor/hooks.json` をコミットします。

3. 生成された git hook が入っていれば、通常の `git commit` をそのまま使えます。
   git hook が使えない環境では `agentnote commit -m "..."` を fallback として使えます。
</Steps>

## セットアップ確認

```bash
agentnote status
```

```text
agentnote v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` は有効な adapter、`capture:` は active な agent hook が何を集めるか、`git:` は managed な repo-local git hook、`commit:` は通常の `git commit` が主経路か fallback mode かを示します。

## 対応状況

| エージェント | 状態 | 補足 |
|---|---|---|
| Claude Code | 対応済み | 行単位の著作判定 |
| Codex CLI | Preview | transcript ベース。patch 行数と commit が一致したときは行単位へ昇格 |
| Cursor | Preview | `afterFileEdit` / `afterTabFileEdit` ベース。最後の AI edit の blob がそのまま commit に残るときだけ安全に line-level へ昇格 |
| Gemini CLI | 近日対応 | |

## Capability の概要

| Capability | Claude Code | Codex CLI | Cursor |
|---|---|---|---|
| 生成された git hook ありの通常 `git commit` | Yes | Yes | Yes |
| `agentnote commit` fallback | Yes | Yes | Yes |
| Prompt / response 復元 | Hook-native | Local transcript | Response hook または local transcript |
| 既定の attribution | Line-level | File-level | File-level |
| 安全な line-level 昇格 | 既定経路 | transcript の patch 行数が commit と一致したとき | `afterFileEdit` / `afterTabFileEdit` の行数が一致し、最後の AI edit の blob がそのまま残るとき |
