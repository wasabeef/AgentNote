## できること

<CardGrid stagger>
  <Card title="AI が書いた行を残す" icon="pencil">
    `git blame` より細かい粒度。Claude Code は行単位で完全、他のエージェントは条件が揃ったとき行単位へ昇格。
  </Card>
  <Card title="プロンプトと返答も一緒に保存" icon="document">
    モデル名・使ったツール情報も含めて、変更とセットで git notes に入れる。
  </Card>
  <Card title="PR に自動でレポート" icon="rocket">
    AI 比率、プロンプト数、コミットハッシュ。レビュアーが最初に見る画面に文脈が載る。
  </Card>
  <Card title="git notes にだけ書く" icon="setting">
    ブランチには混ざらない。push / fetch でチーム共有しつつ、optional な static Dashboard を GitHub Pages に publish できる。
  </Card>
</CardGrid>

## セットアップ

<Steps>
1. リポジトリで init を実行

   ```bash
   npx agent-note init --agent claude
   ```

   - Codex CLI: `npx agent-note init --agent codex`
   - Cursor: `npx agent-note init --agent cursor`
   - Gemini CLI: `npx agent-note init --agent gemini`
   - 複数エージェント同時: `npx agent-note init --agent claude --agent cursor`

2. 生成されたファイルをコミット

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   - Codex CLI: `.codex/config.toml` と `.codex/hooks.json` をコミット
   - Cursor: `.cursor/hooks.json` をコミット
   - Gemini CLI: `.gemini/settings.json` をコミット

3. 生成された git hook が入っていれば、通常の `git commit` をそのまま使えます。
   git hook が使えない環境では `agent-note commit -m "..."` を fallback として使えます。
</Steps>

## セットアップ確認

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` は有効な adapter、`capture:` は active な agent hook が何を集めるか、`git:` は managed な repo-local git hook、`commit:` は通常の `git commit` が主経路か fallback mode かを示します。
