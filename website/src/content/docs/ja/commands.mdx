## agentnote init

リポジトリに Agent Note をセットアップします。

```bash
npx @wasabeef/agentnote init --agent claude
```

| フラグ | 説明 |
|---|---|
| `--no-hooks` | エージェントフックの設定をスキップ |
| `--no-git-hooks` | git フックのインストールをスキップ |
| `--no-action` | ワークフローの作成をスキップ |
| `--no-notes` | 自動 fetch 設定をスキップ |
| `--hooks` | エージェントフックのみインストール |
| `--action` | ワークフローのみ作成 |
| `--agent <name>` | **必須。** エージェントアダプターを指定: `claude`、`codex`、`cursor`、`gemini`。複数エージェントの場合は繰り返し指定 |

## agentnote show

コミットの AI セッション詳細を表示します。

```bash
agentnote show          # HEAD
agentnote show abc1234  # commit SHA
```

## agentnote log

最近のコミットを AI 比率付きで一覧表示します。

```bash
agentnote log      # 直近 10 件
agentnote log 20   # 直近 20 件
```

## agentnote status

現在の tracking state を表示します。

```bash
agentnote status
```

```text
agentnote v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

<Aside>
  `agent:` は有効な adapter、`capture:` は active な agent hook が何を集めるか、`git:` は managed な repo-local git hook、`commit:` は実際の主経路を示します。`commit:` が fallback mode のときは `agentnote commit` を使うのが安全です。
</Aside>

## agentnote deinit

リポジトリから Agent Note のフックと設定を除去します。

```bash
npx @wasabeef/agentnote deinit --agent claude
```

| フラグ | 説明 |
|---|---|
| `--agent <name>` | **必須。** 除去するエージェント: `claude`、`codex`、`cursor`、`gemini`。複数指定可 |
| `--remove-workflow` | `.github/workflows/agentnote.yml` も削除（デフォルトでは温存） |
| `--keep-notes` | notes の自動 fetch 設定を維持 |

<Aside>
  他のエージェントがまだ有効な場合、共有インフラ（git フック、notes 設定）は温存されます。
</Aside>

## agentnote commit

セッショントレーラーと note 記録を含む `git commit` のラッパーです。

```bash
agentnote commit -m "feat: add auth"
```

<Aside type="tip">
  git フックがインストールされていれば不要です — `git commit` が同じ処理を自動的に行います。
</Aside>

<Aside>
  Cursor preview でも、生成された git hook が入っていれば通常の `git commit` をそのまま記録できます。
  現在は prompt、response metadata、`afterFileEdit` / `afterTabFileEdit` による attribution を記録し、最後の AI edit 後の blob がそのまま commit に残っている場合のみ安全に line-level へ昇格します。git hook が使えない環境では `agentnote commit` が fallback として使えます。
</Aside>
