このページは、Agent Note の全体像を把握したあとにコマンドや flag を確認するための reference です。最初の導入手順だけ見たい場合は、先に [インストール](./installation/) を読んでください。

## よく使うコマンド

- 最初のセットアップ: `agent-note init`
- コミットに何が記録されたか見る: `agent-note show`
- その行がなぜ変わったか見る: `agent-note why`
- 最近の活動を一覧する: `agent-note log`
- PR レポートを作る: `agent-note pr`
- hook と有効な adapter を確認する: `agent-note status`

## agent-note init

リポジトリに Agent Note をセットアップします。

```bash
npx agent-note init --agent claude
```

| フラグ | 説明 |
|---|---|
| `--agent <name...>` | **必須。** 1 つ以上のエージェントアダプターを指定: `claude`、`codex`、`cursor`、`gemini` |
| `--dashboard` | 共有 Dashboard 用の `.github/workflows/agentnote-dashboard.yml` も作成 |
| `--no-hooks` | エージェントフックの設定をスキップ |
| `--no-git-hooks` | git フックのインストールをスキップ |
| `--no-action` | ワークフローの作成をスキップ |
| `--no-notes` | 自動 fetch 設定をスキップ |
| `--hooks` | エージェントフックのみインストール |
| `--action` | ワークフローのみ作成 |

## agent-note show

コミットの AI セッション詳細を表示します。

```bash
npx agent-note show          # HEAD
npx agent-note show abc1234  # commit SHA
```

```
commit:  ce941f7 feat: add JWT auth middleware
session: a1b2c3d4-5678-90ab-cdef-111122223333

ai:      60% (45/75 lines) [█████░░░]
model:   claude-sonnet-4-20250514
agent:   claude
files:   5 changed, 3 by AI

  src/middleware/auth.ts  🤖
  src/types/token.ts  🤖
  CHANGELOG.md  👤
```

## agent-note why

指定した行を最後に変更した Commit を `git blame` で調べ、その Commit に紐づく Agent Note context を表示します。

```bash
npx agent-note why src/middleware/auth.ts:42
npx agent-note why src/middleware/auth.ts:42-55
npx agent-note why src/middleware/auth.ts#L42
npx agent-note why @src/middleware/auth.ts#L42
npx agent-note why https://github.com/owner/repo/blob/main/src/middleware/auth.ts#L42
```

<Aside>
  `why` は `path:line`、GitHub 風の `path#Lline`、GitHub file URL、AI Agent の出力からコピーした先頭 `@` 付き path を受け付けます。`git blame` から `git notes --ref=agentnote show <commit>` を手で探す流れを短縮します。
</Aside>

## agent-note log

最近のコミットを AI 比率付きで一覧表示します。

```bash
npx agent-note log      # 直近 10 件
npx agent-note log 20   # 直近 20 件
```

## agent-note pr

PR レポートを生成します。

<Tabs>
  <TabItem label="Markdown">
  ```bash
  npx agent-note pr origin/main
  ```
  </TabItem>
  <TabItem label="Detail">
  ```bash
  npx agent-note pr origin/main --prompt-detail compact
  ```
  </TabItem>
  <TabItem label="JSON">
  ```bash
  npx agent-note pr origin/main --json
  ```
  </TabItem>
  <TabItem label="PR 更新">
  ```bash
  npx agent-note pr origin/main --output description --update 42
  ```
  </TabItem>
</Tabs>

<Aside>
  prompt detail の既定値は `compact` です。`compact` は commit の説明に必要な prompt を中心に表示し、`full` は保存された prompt をすべて表示します。
</Aside>

## agent-note status

現在の tracking 状態を表示します。

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

<Aside>
  `agent:` は有効な adapter、`capture:` は今の hook が何を集めるか、`git:` はリポジトリ単位の git hook の状態、`commit:` は主な記録経路を示します。`commit:` が代替モードのときは `agent-note commit` を使うのが安全です。
</Aside>

## agent-note deinit

リポジトリから Agent Note のフックと設定を除去します。

```bash
npx agent-note deinit --agent claude
```

| フラグ | 説明 |
|---|---|
| `--agent <name...>` | **必須。** 1 つ以上のエージェントを除去: `claude`、`codex`、`cursor`、`gemini` |
| `--remove-workflow` | `.github/workflows/agentnote-pr-report.yml` と `.github/workflows/agentnote-dashboard.yml` も削除 |
| `--keep-notes` | notes の自動 fetch 設定を維持 |

<Aside>
  他のエージェントがまだ有効な場合、共有インフラ（git フック、notes 設定）は温存されます。
</Aside>

## agent-note commit

セッショントレーラーと note 記録を含む `git commit` のラッパーです。

```bash
npx agent-note commit -m "feat: add auth"
```

<Aside type="tip">
  git フックがインストールされていれば不要です — `git commit` が同じ処理を自動的に行います。
</Aside>

<Aside>
  Cursor でも、生成された git hook が入っていれば通常の `git commit` をそのまま記録できます。
  prompt、response metadata、`afterFileEdit` / `afterTabFileEdit` による attribution を記録し、edit 数が一致し、最後の AI edit 後の file がそのまま commit に残っている場合だけ安全に line-level へ昇格します。git hook が使えない環境では `agent-note commit` が代替手段です。
</Aside>
