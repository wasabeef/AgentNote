## インストール

<Steps>
1. **init を実行**

   ```bash
   npx @wasabeef/agentnote init
   ```

   Codex CLI の場合:

   ```bash
   npx @wasabeef/agentnote init --agent codex
   ```

   Cursor の場合:

   ```bash
   npx @wasabeef/agentnote init --agent cursor
   ```

   以下が設定されます:
   - エージェントフック (`.claude/settings.json`、Codex CLI では `.codex/config.toml` と `.codex/hooks.json`、Cursor では `.cursor/hooks.json`)
   - Git フック (`prepare-commit-msg`, `post-commit`, `pre-push`)
   - GitHub Actions ワークフロー (`.github/workflows/agentnote.yml`)
   - notes の自動 fetch 設定

2. **生成ファイルをコミット**

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agentnote"
   git push
   ```

   Codex CLI では `.codex/config.toml` と `.codex/hooks.json` をコミットします。
   Cursor では `.cursor/hooks.json` をコミットします。

3. **完了。** 生成された git hook が入っていれば、Cursor preview を含めて通常の `git commit` で追跡されます。
   git hook が使えない環境では `agentnote commit -m "..."` を fallback として使えます。
</Steps>

## 新しいクローン

各開発者はクローン後に init を実行してローカルの git フックをインストールします:

```bash
git clone <repo> && cd <repo>
npx @wasabeef/agentnote init
```

<Aside>
  Git フックは `.git/` 内に配置されるため、コミットされません。開発者ごとに一度 `init` を実行する必要があります。
</Aside>

## 自動で行われること

init 後は通常の `git commit` で:

1. **Inject** `Agentnote-Session` trailer をコミットメッセージに追加
2. **Record** prompts、responses、file attribution を git note として保存
3. **Push** `git push` 時に notes も自動で送信

Cursor preview では response hook または local transcript による prompt / response 復元も行われます。
git hook が使えない環境では `agentnote commit` が fallback です。

```bash
agentnote status
```

```text
agentnote v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` は有効な adapter、`capture:` は active な agent hook が何を集めるか、`git:` は managed な repo-local git hook、`commit:` は通常の `git commit` が主経路か、fallback を使うべきかを示します。

## エージェント別 capability

| Capability | Claude Code | Codex CLI | Cursor |
|---|---|---|---|
| 生成された git hook ありの通常 `git commit` | Yes | Yes | Yes |
| `agentnote commit` fallback | Yes | Yes | Yes |
| Prompt / response 復元 | Hook-native | Local transcript | Response hook または local transcript |
| 既定の attribution | Line-level | File-level | File-level |
| 安全な line-level 昇格 | 既定経路 | transcript の patch 行数が commit と一致したとき | `afterFileEdit` / `afterTabFileEdit` の行数が一致し、最後の AI edit の blob がそのまま残るとき |
