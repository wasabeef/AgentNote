## クイックセットアップ

`agent-note init` で自動生成されるワークフローを追加:

```yaml
name: Agent Note
on:
  pull_request:
    types: [opened, synchronize]
permissions:
  contents: read
  pull-requests: write
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: wasabeef/AgentNote@v0
```

これだけです。すべての PR に AI セッションレポートが投稿されます。

## 入力

| 入力 | デフォルト | 説明 |
|---|---|---|
| `base` | PR のベースブランチ | 比較対象のブランチ |
| `pr_output` | `description` | PR レポートの出力先: `description` / `comment` / `none` |
| `dashboard` | `false` | 現在の PR 用の dashboard snapshot を書き出す |
| `dashboard_dir` | `packages/dashboard/public/view-demo` | `notes/*.json` の出力先 |
| `output` | — | レガシー: `pr_output` の旧名 |
| `comment` | `true` | レガシー: `false` で PR レポート投稿を無効化 |

### PR Description と dashboard を同時に使う

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
    dashboard_dir: packages/dashboard/public/view-demo
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## 出力

| 出力 | 型 | 説明 |
|---|---|---|
| `overall_ai_ratio` | number | PR 全体の AI 比率 (0-100) |
| `overall_method` | string | `line`, `file`, `mixed`, `none` |
| `tracked_commits` | number | agent-note データ付きのコミット数 |
| `total_commits` | number | PR 内の全コミット数 |
| `total_prompts` | number | 全プロンプト数 |
| `json` | string | 構造化レポート全体 |
| `markdown` | string | レンダリング済みのマークダウンレポート |
| `dashboard_dir` | string | dashboard bundle の出力先ディレクトリ |
| `dashboard_commits` | number | dashboard bundle に書き出した tracked commit 数 |

<Aside type="tip">
  レポート内のコミットハッシュは GitHub のコミットページにリンクされます。
</Aside>

## Dashboard Package

interactive な dashboard は `packages/dashboard/` に分離されています。Action はその package 配下の data bundle を更新し、実際の GitHub Pages への publish は workflow 側で制御できます。

## GitHub Pages Sync

live dashboard は `pull_request` 専用よりも `push` 主体の workflow にしておくのがおすすめです。

- PR branch への `push` 時点で commit ごとの note を保存できるので、`squash merge` 前の SHA を取りこぼしません
- 永続データは `gh-pages/dashboard/notes/*.json` にだけ置きます
- branch push 時は note data だけ更新し、dashboard 本体は現在の `main` shell を使い続けられます
- `main` への `push` で、対応する PR の note を `merged` として確定できます

推奨フロー:

1. `gh-pages` branch から `dashboard/notes` を復元する
2. branch push では `owner:branch` に対応する open PR を引き、同じ PR 番号の既存 note を一度消してから、現在の PR commit だけを書き直す
3. `main` push では対応する PR note を `merged` に更新し、direct push の note も追加する
4. その状態で Pages artifact を build / publish する

これで `main` を JSON で汚さずに、1 つの shared dashboard から open / merged 両方の PR 履歴を見られます。
