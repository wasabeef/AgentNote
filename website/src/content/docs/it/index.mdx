## Configurazione in 60 secondi

<Steps>
1. Esegui init nel tuo repository

   ```bash
   npx @wasabeef/agentnote init
   ```

   Per Codex CLI:

   ```bash
   npx @wasabeef/agentnote init --agent codex
   ```

   Per Cursor:

   ```bash
   npx @wasabeef/agentnote init --agent cursor
   ```

2. Committa i file generati

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agentnote"
   git push
   ```

   I repository Codex versionano `.codex/config.toml` e `.codex/hooks.json` al posto di `.claude/settings.json`.
   I repository Cursor versionano `.cursor/hooks.json`.

3. Claude Code e Codex possono usare `git commit` normalmente.
   Con i git hook generati, Cursor preview puo anche usare il normale `git commit`.
   `agentnote commit -m "..."` resta il fallback quando i git hook non sono disponibili.
</Steps>

## Supporto

| Agente | Stato | Note |
|---|---|---|
| Claude Code | Supporto completo | Attribuzione riga per riga tramite coppie di blob catturate dagli hook |
| Codex CLI | Preview | Basato su transcript; sale al livello di riga quando i conteggi di patch coincidono con il commit |
| Cursor | Preview | Attribuzione basata su `afterFileEdit` / `afterTabFileEdit`, con upgrade sicuro al livello di riga quando il blob finale coincide ancora con l'edit dell'IA |
| Gemini CLI | Prossimamente | |
