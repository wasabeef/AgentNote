## Cosa ottieni

Il Dashboard offre al team un’unica URL condivisa per consultare la cronologia di Agent Note.

- PR aperte e mergeate in un solo posto
- Commit, prompt e attribution senza aprire ogni singola PR
- Un’unica URL GitHub Pages, di solito sotto `/dashboard/`

Usalo quando il PR report non basta più e vuoi una vista condivisa per il team.

## Prima di iniziare

Ti serve il Dashboard solo se vuoi una vista web condivisa.

- Se i report di sessione IA dentro le pull request ti bastano, fermati a [GitHub Action](./github-action/).
- Se vuoi una vista Pages condivisa per la cronologia aperta e mergeata, continua qui.

## Setup rapido

1. Run Dashboard setup once.

```bash
npx agent-note init --agent claude --dashboard
```

2. Commit both workflow files.

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. In GitHub Pages, choose **GitHub Actions** as the source.

The generated Dashboard workflow restores data from `gh-pages`, refreshes it from `refs/notes/agentnote`, and publishes the shared `/dashboard/` view.

## URL pubblica

Di solito il Dashboard viene pubblicato qui:

- `https://<owner>.github.io/<repo>/dashboard/`

Per esempio:

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Se pubblichi con GitHub Pages, il workflow Pages passa i valori specifici del repository per `SITE` e `BASE` al momento del build. Nella maggior parte dei repository puoi usare il workflow di esempio senza modificare `packages/dashboard`.
</Aside>

## Primo deploy

Anche un repository nuovo può accumulare dati del Dashboard prima del primo deploy in produzione. L’URL pubblica di Pages compare però solo dopo il primo deploy su `default branch`.

## Come funziona

- `refs/notes/agentnote` remains the source of truth
- The generated Dashboard workflow writes shared snapshots to `gh-pages/dashboard/notes/*.json`
- GitHub Pages serves the built Dashboard at `/dashboard/`

The repository does not commit sample Dashboard JSON. A fresh Dashboard stays empty until the workflow restores or generates note files. `pull_request` refreshes open history, and `default branch` publishes the merged state.
