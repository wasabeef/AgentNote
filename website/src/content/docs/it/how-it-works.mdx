Quando i git hook generati sono installati, nella maggior parte dei repository basta continuare a usare `git commit` e `git push`. Cursor Preview recupera anche le coppie prompt / response dai transcript locali e può continuare a tracciare i commit quando i git hook non sono disponibili.

### File di sessione

| File | Scopo |
|---|---|
| `prompts.jsonl` | Un prompt per riga con numero di turn |
| `changes.jsonl` | File toccati dall’IA con post-edit blob hash |
| `pre_blobs.jsonl` | Stato del file prima di ogni edit dell’IA |
| `heartbeat` | Timestamp di attività della sessione |
| `turn` | Contatore monotono incrementato a ogni prompt |

### Note Schema

```json
{
  "v": 1,
  "agent": "claude",
  "session_id": "a1b2c3d4-...",
  "model": "claude-sonnet-4-20250514",
  "interactions": [
    {
      "prompt": "Implementa middleware di autenticazione JWT",
      "response": "Creerò il middleware...",
      "files_touched": ["src/auth.ts"],
      "tools": ["Edit"]
    }
  ],
  "files": [
    { "path": "src/auth.ts", "by_ai": true },
    { "path": "CHANGELOG.md", "by_ai": false }
  ],
  "attribution": {
    "ai_ratio": 73,
    "method": "line",
    "lines": { "ai_added": 146, "total_added": 200, "deleted": 12 }
  }
}
```

<Aside type="tip">
  Le liste di prompt arrivano dalla conversazione che ha portato al diff finale, non da tutto il backlog della sessione. In modalità file-level, Agent Note conserva solo il percorso dell’ultimo edit e il contesto vicino che lo spiega.
</Aside>

<Aside>
  I generated artifact più comuni vengono esclusi dal denominatore di `AI ratio` in best effort, inclusi gli output di build web, i Flutter registrants, il Dart codegen, i Go protobuf, i Rust bindings e i generated source Swift / Kotlin.
</Aside>

## Ciclo di vita della sessione

| Evento | Cosa succede |
|---|---|
| **SessionStart** | Crea la directory della sessione e scrive l’heartbeat |
| **UserPromptSubmit** | Appende il prompt, incrementa il contatore di turn e ruota i log |
| **PreToolUse Edit/Write** | Cattura in modo sincrono il pre-edit blob hash |
| **PostToolUse Edit/Write** | Cattura il post-edit blob hash e il percorso del file |
| **Stop** | Registra l’evento stop (l’heartbeat resta attivo — Stop = fine della response, non fine della sessione) |

<Aside>
  I git hook verificano la freschezza della sessione tramite heartbeat prima di inserire trailer o registrare notes. L’heartbeat viene aggiornato su SessionStart e a ogni prompt.
</Aside>

<Aside type="tip">
  Agent Note tiene traccia dell’agente che ha prodotto la sessione. I metadati della sessione salvano l’adapter attivo, così `show`, `status` e la registrazione del commit possono risolvere in seguito il formato corretto del transcript.
</Aside>

## Git Notes

Le git notes sono invisibili nelle liste dei branch, nell’interfaccia GitHub e in CI — ma possono essere pushate e fetchate.

```bash
# Condividi con il team
git push origin refs/notes/agentnote

# Recupera dal remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# Mostra una note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` configura l’auto-fetch e l’hook `pre-push` gestisce l’auto-push. Non servono comandi manuali.
</Aside>
