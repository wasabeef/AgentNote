O Agent Note grava o mesmo formato de git note para todos os agents, mas cada agent expõe hooks e transcripts diferentes. Esta página mostra o que esperar antes de ativar para um time.

## Matriz de capacidades dos agents

| Agent | Status | Prompt | Response | Files | AI Ratio | Line Estimate |
|---|---|---|---|---|---|---|
| Claude Code | Full support | Sim | Sim | Sim | Sim | Por padrão |
| Codex CLI | Supported | Sim | Sim | Sim | Sim | Quando o histórico de patches do Codex bate com o Commit final |
| Cursor | Supported | Sim | Sim | Sim | Sim | Quando a contagem de edições e o arquivo final coincidem |
| Gemini CLI | Preview | Sim | Sim | Sim | Sim | Ainda não |

<Aside>
  Files significa que Agent Note pode mostrar quais arquivos commitados foram tocados pelo Agent. Line Estimate significa que ele também pode estimar linhas escritas pela IA, em vez de apenas contar arquivos. Preview significa que o Agent Note acompanha o comportamento público atual, mas o formato local do agent ainda pode mudar.
</Aside>

## Setup recomendado

```bash
npx agent-note init --agent claude
```

Ative apenas os agents que o time usa de fato. Vários agents podem coexistir no mesmo repository.

```bash
npx agent-note init --agent claude codex cursor gemini
```

Cada developer precisa rodar `init` uma vez depois do clone, porque git hooks ficam dentro de `.git/`.

## Escolha pela expectativa

| Você precisa de... | Espere... |
|---|---|
| A attribution por linha mais completa hoje | Claude Code |
| Suporte Codex CLI com attribution conservadora | Codex CLI |
| Histórico do editor Cursor e recuperação de response | Cursor |
| Captura de prompt / response no Gemini CLI | Gemini CLI |

## Attribution

Line-level conta AI-added lines quando o Agent Note consegue comparar com segurança o AI edit, edits humanos posteriores e o final commit diff. File-level é usado quando a prova não é suficiente.

O Agent Note prefere fallback conservador a adivinhar.

## Notas por agent

### Claude Code

Claude Code fornece prompts, responses e edits por hooks. O Agent Note salva o estado do arquivo antes e depois do AI edit e remove edits humanos posteriores do diff final.

### Codex CLI

Codex CLI é transcript-driven. O Agent Note lê transcripts locais e sobe para line-level apenas quando os counts de `apply_patch` batem com o diff final.

### Cursor

Cursor usa `afterFileEdit` / `afterTabFileEdit`, além de response hooks ou transcripts locais. Line-level só é possível quando a contagem de edições e o arquivo final ainda coincidem com a última edição da IA.

### Gemini CLI

Gemini CLI usa `BeforeTool` / `AfterTool` para arquivos e `BeforeAgent` / `AfterAgent` para prompt / response. A attribution atual é file-level.

## Quando faltam dados

- Nenhuma note é escrita quando o Agent Note não encontra arquivo commitado tocado pelo agent.
- Um prompt antigo pode ser omitido se pertencer a outra commit window.
- Uma response pode faltar se o agent não a expõe.
- Artefatos gerados e caminhos listados na `.agentnoteignore` na raiz do repositório continuam visíveis em Notes, PR Report e Dashboard; eles são excluídos apenas do denominador do AI ratio.

Para investigar, comece por [Troubleshooting](./troubleshooting/).
