### Arquivos da sessão

| Arquivo | Finalidade |
|---|---|
| `prompts.jsonl` | Um prompt por linha com número de turn |
| `changes.jsonl` | Arquivos tocados pela IA com post-edit blob hashes |
| `pre_blobs.jsonl` | Estado do arquivo antes de cada edição da IA |
| `heartbeat` | Timestamp de atividade da sessão |
| `turn` | Contador monotônico incrementado a cada prompt |

## Modos de atribuição

O Claude Code calcula autoria de IA por linha usando um algoritmo de posição com 3 diffs:

| Diff | De | Para | O que captura |
|---|---|---|---|
| diff1 | commit pai | arquivo committed | Todas as mudanças deste commit |
| diff2 | antes da edição da IA | arquivo committed | Edição da IA + edições humanas posteriores |
| diff3 | depois da edição da IA | arquivo committed | Apenas edições humanas |

**Posições da IA = posições do diff2 que NÃO estão no diff3**

Todos os três diffs apontam para o committed blob, então as posições são diretamente comparáveis.

<Aside type="tip">
  Deleções são acompanhadas separadamente e não são atribuídas nem à IA nem ao humano. Quando os dados de blob não estão disponíveis, o Agent Note recua automaticamente para atribuição por arquivo.
</Aside>

O Codex CLI usa atribuição guiada por transcript. Quando os contadores de linha de `apply_patch` no transcript batem com o diff final do commit, o Agent Note promove commits do Codex para `method: "line"`. Caso contrário, ele volta para `method: "file"` para evitar falsos positivos. Edições shell-only que nunca aparecem como patch data no transcript permanecem do lado seguro: o Agent Note ainda pode preservar o par prompt / response e os tools do Codex observados, mas não adivinha `files_touched` nem AI-authored files apenas a partir de passos de shell. Esses commits, portanto, ficam em `method: "file"` com `0%` de atribuição de IA, a menos que outro primary signal conecte arquivos ao agente. Se o transcript não puder ser lido no momento do commit, o Agent Note emite um warning e pula a criação da note em vez de gravar dados de atribuição incertos.

O Cursor preview usa os hooks `afterFileEdit` e `afterTabFileEdit` para atribuição. Quando as contagens de edição batem e o committed blob final ainda corresponde à última edição da IA, o Agent Note promove esses arquivos com segurança para `method: "line"`. Caso contrário, ele volta para `method: "file"`, enquanto continua recuperando pares de prompt / response dos response hooks do Cursor ou de transcripts locais quando disponíveis.

O Gemini CLI preview usa hooks `BeforeTool`/`AfterTool` sobre as ferramentas `write_file` e `replace` para atribuição em nível de arquivo. A detecção de commit usa o padrão pending-commit via `BeforeTool`/`AfterTool` em `run_shell_command`. A recuperação de prompt e response vem dos hooks `BeforeAgent`/`AfterAgent` e do parsing de transcripts JSONL.

<Aside type="tip">
  As listas de prompts vêm da conversa causal em torno do diff final, não de todo o histórico da sessão. Quando o Agent Note faz fallback para `method: "file"`, ele ancora essa janela no latest touch turn de cada committed file e mantém apenas o prompt-only context que leva a esses edits finais. Same-file edits antigas e follow-up chatter depois do edit ficam de fora. O Agent Note também exclui, na medida do possível, artefatos gerados comuns do denominador de `AI ratio`, incluindo outputs de build web, Flutter registrants, Dart codegen, protobufs de Go, bindings de Rust e fontes geradas de Swift / Kotlin.
</Aside>

## Ciclo de vida da sessão

| Evento | O que acontece |
|---|---|
| **SessionStart** | Cria o diretório da sessão e grava o heartbeat |
| **UserPromptSubmit** | Acrescenta o prompt, incrementa o contador de turn e rotaciona os logs |
| **PreToolUse Edit/Write** | Captura o pre-edit blob hash de forma síncrona |
| **PostToolUse Edit/Write** | Captura o post-edit blob hash e o caminho do arquivo |
| **Stop** | Registra o evento stop (o heartbeat continua ativo — Stop = fim da response, não fim da sessão) |

<Aside>
  Os git hooks verificam a atualidade da sessão pelo heartbeat antes de injetar trailers ou gravar notes. O heartbeat é atualizado em SessionStart e em cada prompt.
</Aside>

<Aside type="tip">
  O Agent Note entende qual agent está ativo. Os metadados da sessão armazenam o adapter ativo, então `show`, `status` e o registro do commit conseguem resolver o formato correto do transcript depois.
</Aside>

## Git Notes

Git notes são invisíveis em listas de branches, na UI do GitHub e no CI, mas podem ser pushadas e fetchadas.

```bash
# Compartilhar com o time
git push origin refs/notes/agentnote

# Buscar do remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# Ver uma note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` configura o auto-fetch, e o hook `pre-push` cuida do auto-push. Nenhum comando manual é necessário.
</Aside>
