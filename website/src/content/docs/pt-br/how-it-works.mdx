Quando os git hooks gerados estão instalados, na maioria dos repositórios basta continuar usando `git commit` e `git push`. O Cursor Preview também recupera pares de prompt / response a partir de transcripts locais e consegue continuar registrando commits quando os git hooks não estão disponíveis.

### Arquivos da sessão

| Arquivo | Finalidade |
|---|---|
| `prompts.jsonl` | Um prompt por linha com número de turn |
| `changes.jsonl` | Arquivos tocados pela IA com post-edit blob hashes |
| `pre_blobs.jsonl` | Estado do arquivo antes de cada edição da IA |
| `heartbeat` | Timestamp de atividade da sessão |
| `turn` | Contador monotônico incrementado a cada prompt |

## Como o `📝 Context` é coletado

`📝 Context` é metadata apenas visual para prompts curtos como "continue" ou "faça o próximo". Ele ajuda a entender o prompt sem mudar a attribution.

<img src="/AgentNote/images/context-dashboard-example.png" alt="Agent Note Dashboard showing Context before a short prompt" style="width: 100%; border-radius: 20px; border: 1px solid rgba(148, 163, 184, 0.18); box-shadow: 0 24px 80px rgba(15, 23, 42, 0.28);" />

- O `reference` context vem da response imediatamente anterior quando essa response contém um structural anchor forte, como changed file path, file basename ou code identifier do diff final.
- O `scope` context vem da response atual quando o começo dela nomeia o escopo do trabalho com structural anchors.
- O Agent Note não usa approval keywords como `yes`, `continue` ou `お願いします` como regra de seleção. Se não houver structural anchor, ele omite Context.
- Context nunca muda `files_touched`, prompt ownership, AI ratio, attribution method ou se uma git note será gravada.

## Agent Note vs Entire

Agent Note e Entire resolvem problemas relacionados, mas diferentes.

- Agent Note conecta git commits à AI conversation que os produziu. Entire é um checkpoint product mais amplo com workflows de rewind / resume.
- Agent Note guarda o registro permanente em `refs/notes/agentnote`; arquivos temporários da sessão ficam em `.git/agentnote/`. Entire é centrado em checkpoint metadata e uma hosted web application.
- O Dashboard do Agent Note é static GitHub Pages opcional, apoiado por git notes. Você pode continuar usando Agent Note sem hosted service.
- Agent Note não tenta restaurar um workspace state antigo. Ele foca em reviewability: por que um commit mudou, quais prompts participaram e quanto do diff foi AI-authored.

<Aside type="tip">
  As listas de prompts vêm da janela de conversa entre commits, não de todo o backlog da sessão. O Agent Note mantém prompts úteis de planejamento, esclarecimento e revisão do commit atual, e remove históricos de prompts citados e turnos de edição que não sobreviveram neste commit.
</Aside>

<Aside>
  Generated artifacts comuns são excluídos do denominador de `AI ratio` em best effort usando sinais específicos, como caminhos de framework/cache, nomes ou sufixos gerados e headers dos arquivos commitados. Nomes genéricos de diretório como `build/`, `dist/`, `gen/`, `generated/` e `target/` não bastam por si só.
</Aside>

## Ciclo de vida da sessão

| Evento | O que acontece |
|---|---|
| **SessionStart** | Cria o diretório da sessão e grava o heartbeat |
| **UserPromptSubmit** | Acrescenta o prompt, incrementa o contador de turn e rotaciona os logs |
| **PreToolUse Edit/Write** | Captura o pre-edit blob hash de forma síncrona |
| **PostToolUse Edit/Write** | Captura o post-edit blob hash e o caminho do arquivo |
| **Stop** | Registra o evento stop (o heartbeat continua ativo — Stop = fim da response, não fim da sessão) |

<Aside>
  Os git hooks verificam a atualidade da sessão pelo heartbeat antes de injetar trailers ou gravar notes. O heartbeat é atualizado em SessionStart e em cada prompt.
</Aside>

<Aside type="tip">
  O Agent Note entende qual agent está ativo. Os metadados da sessão armazenam o adapter ativo, então `show`, `status` e o registro do commit conseguem resolver o formato correto do transcript depois.
</Aside>

## Git Notes

Git notes são invisíveis em listas de branches, na UI do GitHub e no CI, mas podem ser pushadas e fetchadas.

```bash
# Compartilhar com o time
git push origin refs/notes/agentnote

# Buscar do remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# Ver uma note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` configura o auto-fetch, e o hook `pre-push` cuida do auto-push. Nenhum comando manual é necessário.
</Aside>
