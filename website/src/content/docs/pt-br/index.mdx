## O que o Agent Note faz

<CardGrid stagger>
  <Card title="Atribuição de IA por linha" icon="pencil">
    Claude Code tem atribuição completa por linha. O Codex sobe para nível de linha quando as contagens de patch do transcript batem com o commit final.
  </Card>
  <Card title="Rastreamento de sessão" icon="document">
    Prompts, respostas, nome do modelo e uso de ferramentas — tudo vinculado aos commits que geraram.
  </Card>
  <Card title="Relatórios de PR" icon="rocket">
    Relatórios automáticos de sessões de IA em pull requests com proporção por commit, contagem de linhas e hashes vinculados.
  </Card>
  <Card title="Nativo do Git" icon="setting">
    Armazenado como git notes — invisível para branches e CI. Pode ser enviado, buscado e compartilhado.
  </Card>
</CardGrid>

## Configuração em 60 segundos

<Steps>
1. Execute init no seu repositório

   ```bash
   npx agent-note init --agent claude
   ```

   Para Codex CLI: `npx agent-note init --agent codex`
   Para Cursor: `npx agent-note init --agent cursor`
   Para Gemini CLI: `npx agent-note init --agent gemini`
   Vários agentes ao mesmo tempo: `npx agent-note init --agent claude --agent cursor`

2. Faça commit dos arquivos gerados

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   Repositórios com Codex versionam `.codex/config.toml` e `.codex/hooks.json` no lugar de `.claude/settings.json`.
   Repositórios com Cursor versionam `.cursor/hooks.json`.
   Repositórios Gemini CLI fazem commit de `.gemini/settings.json`.

3. Claude Code e Codex podem usar `git commit` normalmente.
   Com os git hooks gerados, o Cursor preview também pode usar `git commit` normalmente.
   Use `agent-note commit -m "..."` como fallback quando os git hooks não estiverem disponíveis.
</Steps>

## Compatibilidade

| Agente | Status | Notas |
|---|---|---|
| Claude Code | Suporte completo | Atribuição por linha via pares de blobs capturados pelos hooks |
| Codex CLI | Preview | Baseado em transcript; sobe para nível de linha quando as contagens de patch batem com o commit |
| Cursor | Preview | Atribuição baseada em `afterFileEdit` / `afterTabFileEdit`, com upgrade seguro para nível de linha quando o blob final ainda corresponde à edição da IA |
| Gemini CLI | Preview | Atribuição de arquivos via hooks `BeforeTool`/`AfterTool` em `.gemini/settings.json` |
