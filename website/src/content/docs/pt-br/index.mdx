## Configuração

<Steps>
1. Execute init no seu repositório

   ```bash
   npx agent-note init --agent claude
   ```

   - Para Codex CLI: `npx agent-note init --agent codex`
   - Para Cursor: `npx agent-note init --agent cursor`
   - Para Gemini CLI: `npx agent-note init --agent gemini`
   - Vários agentes ao mesmo tempo: `npx agent-note init --agent claude cursor`

2. Faça commit dos arquivos gerados

   ```bash
   git add .claude/settings.json .github/workflows/agentnote-pr-report.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   - Codex CLI: versionar `.codex/config.toml` e `.codex/hooks.json`
   - Cursor: versionar `.cursor/hooks.json`
   - Gemini CLI: versionar `.gemini/settings.json`

3. Codifique com IA.
   Quando os git hooks gerados estão instalados, basta continuar usando `git commit`.
</Steps>

## Verifique sua configuração

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` mostra o adaptador ativo, `capture:` resume o que os hooks ativos coletam, `git:` mostra se os git hooks gerenciados estão instalados, e `commit:` indica se o `git commit` normal é o caminho principal.
