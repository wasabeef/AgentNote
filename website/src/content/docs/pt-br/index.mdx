## O que faz

<CardGrid stagger>
  <Card title="Manter registro das linhas escritas pela IA" icon="pencil">
    Mais fino que `git blame`. Claude Code é completo no nível de linha; outros agentes são promovidos quando as condições correspondem.
  </Card>
  <Card title="Prompts e respostas salvos junto" icon="document">
    Nome do modelo e uso de ferramentas também — tudo agrupado em git notes com a mudança.
  </Card>
  <Card title="Relatórios automáticos de PR" icon="rocket">
    Proporção de IA, contagem de prompts, hashes de commits. Revisores veem o contexto antes do código.
  </Card>
  <Card title="Escrito apenas em git notes" icon="setting">
    Branches ficam limpos. Push / fetch para compartilhar com o time.
  </Card>
</CardGrid>

## Configuração

<Steps>
1. Execute init no seu repositório

   ```bash
   npx agent-note init --agent claude
   ```

   - Para Codex CLI: `npx agent-note init --agent codex`
   - Para Cursor: `npx agent-note init --agent cursor`
   - Para Gemini CLI: `npx agent-note init --agent gemini`
   - Vários agentes ao mesmo tempo: `npx agent-note init --agent claude --agent cursor`

2. Faça commit dos arquivos gerados

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   - Codex CLI: versionar `.codex/config.toml` e `.codex/hooks.json`
   - Cursor: versionar `.cursor/hooks.json`
   - Gemini CLI: versionar `.gemini/settings.json`

3. Codifique com IA.
   O `git commit` normal funciona quando os git hooks gerados estão instalados.
   `agent-note commit -m "..."` continua sendo o fallback quando os git hooks não estão disponíveis.
</Steps>

## Verifique sua configuração

```bash
npx agent-note status
```

```text
agent-note v0.x.x

agent:   active (cursor)
capture: cursor(prompt, response, edits, shell)
git:     active (prepare-commit-msg, post-commit, pre-push)
commit:  tracked via git hooks
session: a1b2c3d4…
agent:   cursor
linked:  3/20 recent commits
```

`agent:` mostra o adaptador habilitado, `capture:` resume o que os hooks do agente ativo coletam, `git:` mostra se os git hooks locais do repositório estão instalados, e `commit:` indica se o `git commit` normal é o caminho principal ou se o modo fallback está ativo.
