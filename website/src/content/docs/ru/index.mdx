## Что делает Agent Note

<CardGrid stagger>
  <Card title="Атрибуция ИИ на уровне строк" icon="pencil">
    Claude Code даёт полную атрибуцию на уровне строк. Codex повышается до уровня строк, когда счётчики patch в transcript совпадают с финальным коммитом.
  </Card>
  <Card title="Отслеживание сессий" icon="document">

  </Card>
  <Card title="Отчёты в PR" icon="rocket">

## Настройка за 60 секунд

<Steps>
1. Запустите init в вашем репозитории

   ```bash
   npx agent-note init --agent claude
   ```

   Для Codex CLI: `npx agent-note init --agent codex`
   Для Cursor: `npx agent-note init --agent cursor`
   Для Gemini CLI: `npx agent-note init --agent gemini`
   Несколько агентов одновременно: `npx agent-note init --agent claude --agent cursor`

2. Закоммитьте сгенерированные файлы

   ```bash
   git add .claude/settings.json .github/workflows/agentnote.yml
   git commit -m "chore: enable agent-note"
   git push
   ```

   Репозитории с Codex коммитят `.codex/config.toml` и `.codex/hooks.json` вместо `.claude/settings.json`.
   Репозитории с Cursor коммитят `.cursor/hooks.json`.
   Для Gemini CLI коммитьте `.gemini/settings.json`.

3. Claude Code и Codex могут использовать обычный `git commit`.
   С установленными сгенерированными git hooks Cursor preview также может использовать обычный `git commit`.
   `agent-note commit -m "..."` остаётся fallback-вариантом, если git hooks недоступны.
</Steps>

## Поддержка

| Агент | Статус | Примечания |
|---|---|---|
| Claude Code | Полная поддержка | Построчная атрибуция через пары blob, сохранённые хуками |
| Codex CLI | Preview | Основано на transcript; повышается до уровня строк, когда счётчики patch совпадают с коммитом |
| Cursor | Preview | Атрибуция через `afterFileEdit` / `afterTabFileEdit` с безопасным повышением до строк, когда итоговый blob всё ещё совпадает с AI-правкой |
| Gemini CLI | Preview | Атрибуция на уровне файлов через хуки `BeforeTool`/`AfterTool` в `.gemini/settings.json` |
