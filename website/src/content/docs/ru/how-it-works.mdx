Эта страница объясняет механику отчета: что собирается локально, что превращается в git note и как эти данные попадают в PR Report и Dashboard. Чтобы пользоваться Agent Note, не нужно помнить все детали, но это полезная модель, когда результат выглядит неожиданно.

## Хранилище

<CardGrid>
  <Card title="Слой 1: локальные временные данные" icon="laptop">
    Данные сессии в `.git/agentnote/sessions/`. JSONL-файлы для prompts, changes и blob hashes. Ротируются после каждого commit и никогда не pushатся.
  </Card>
  <Card title="Слой 2: Git Notes" icon="document">
    Постоянная запись в `refs/notes/agentnote`. Один JSON на commit. Можно push, fetch и делиться с командой.
  </Card>
</CardGrid>

### Файлы сессии

| Файл | Назначение |
|---|---|
| `prompts.jsonl` | Один prompt на строку с номером turn |
| `changes.jsonl` | Файлы, которых коснулась AI, с post-edit blob hashes |
| `pre_blobs.jsonl` | Состояние файла до каждого AI edit |
| `heartbeat` | Timestamp активности сессии |
| `turn` | Монотонный счетчик, увеличивающийся с каждым prompt |

## Как собирается `📝 Context`

`📝 Context` — это подсказка только для отображения коротких prompt вроде "continue" или "сделай следующий шаг". Она помогает понять prompt, не меняя attribution.

<img src="/AgentNote/images/context-dashboard-example.png" alt="Agent Note Dashboard showing Context before a short prompt" style="width: 100%; border-radius: 20px; border: 1px solid rgba(148, 163, 184, 0.18); box-shadow: 0 24px 80px rgba(15, 23, 42, 0.28);" />

- `reference` context берётся из непосредственно предыдущего response, если он упоминает путь изменённого файла, имя файла или code identifier из финального diff.
- `scope` context берётся из текущего response, если его первые строки ясно описывают текущую работу.
- Agent Note не использует короткие подтверждения вроде `yes`, `continue` или `お願いします` как правила выбора. Если нет конкретной подсказки по файлу или коду, Context не добавляется.
- Context никогда не меняет `files_touched`, prompt ownership, AI ratio, attribution method или факт сохранения git note.

## Agent Note vs Entire

Agent Note и Entire решают похожие, но разные задачи.

- Agent Note связывает git commits с AI conversation, которая их создала. Entire — более широкий checkpoint product с workflow rewind / resume.
- Agent Note хранит постоянную запись в `refs/notes/agentnote`; временные файлы сессии остаются в `.git/agentnote/`. Entire строится вокруг checkpoint metadata и hosted web application.
- Dashboard Agent Note — это optional static GitHub Pages на основе git notes. Agent Note можно использовать без hosted service.
- Agent Note не пытается восстановить старый workspace state. Он фокусируется на reviewability: почему commit изменился, какие prompts участвовали и какая часть diff была AI-authored.

<Aside type="tip">
  Списки prompt берутся из окна разговора между commit, а не из всей истории сессии. Agent Note сохраняет полезные prompt для планирования, уточнения и review текущего commit, но убирает процитированную старую историю prompt и edit turn, которые не попали в этот commit.
</Aside>

<Aside>
  Распространённые generated artifacts исключаются из знаменателя `AI ratio` в best effort по конкретным сигналам: путям framework/cache, generated именам или суффиксам файлов и заголовкам закоммиченных файлов. Общих имён директорий вроде `build/`, `dist/`, `gen/`, `generated/` и `target/` самих по себе недостаточно. Если repository коммитит bundles или generated outputs, которые должны оставаться видимыми, но не влиять на AI Ratio, добавьте их в файл `.agentnoteignore` в корне репозитория.
</Aside>

## Жизненный цикл сессии

| Событие | Что происходит |
|---|---|
| **SessionStart** | Создаёт директорию сессии и записывает heartbeat |
| **UserPromptSubmit** | Добавляет prompt, увеличивает turn counter и ротирует logs |
| **PreToolUse Edit/Write** | Синхронно захватывает pre-edit blob hash |
| **PostToolUse Edit/Write** | Захватывает post-edit blob hash и путь к файлу |
| **Stop** | Логирует событие stop (heartbeat остаётся активным — Stop = конец response, а не конец сессии) |

<Aside>
  Git hooks проверяют свежесть сессии по heartbeat и наличие записываемых session data перед вставкой trailers или записью notes. Heartbeat обновляется hook events во время длинных turns; metadata-only sessions пропускаются. Один путь к transcript — это только вспомогательные metadata, а не записываемые data.
</Aside>

<Aside type="tip">
  Agent Note знает, какой agent активен. Метаданные сессии хранят текущий adapter, поэтому `show`, `status` и запись commit позже могут определить правильный формат transcript.
</Aside>

## Git Notes

Git notes невидимы в списках веток, в UI GitHub и в CI, но их можно push и fetch.

```bash
# Поделиться с командой
git push origin refs/notes/agentnote

# Забрать из remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# Посмотреть note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` настраивает auto-fetch, а hook `pre-push` отвечает за auto-push. Ручные команды не нужны.
</Aside>
