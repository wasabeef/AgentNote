## Хранилище

<CardGrid>
  <Card title="Слой 1: локальные временные данные" icon="laptop">
    Данные сессии в `.git/agentnote/sessions/`. JSONL-файлы для prompts, changes и blob hashes. Ротируются после каждого commit и никогда не pushатся.
  </Card>
  <Card title="Слой 2: Git Notes" icon="document">
    Постоянная запись в `refs/notes/agentnote`. Один JSON на commit. Можно push, fetch и делиться с командой.
  </Card>
</CardGrid>

### Файлы сессии

| Файл | Назначение |
|---|---|
| `prompts.jsonl` | Один prompt на строку с номером turn |
| `changes.jsonl` | Файлы, которых коснулась AI, с post-edit blob hashes |
| `pre_blobs.jsonl` | Состояние файла до каждого AI edit |
| `heartbeat` | Timestamp активности сессии |
| `turn` | Монотонный счетчик, увеличивающийся с каждым prompt |

### Note Schema

```json
{
  "v": 1,
  "agent": "claude",
  "session_id": "a1b2c3d4-...",
  "model": "claude-sonnet-4-20250514",
  "interactions": [
    {
      "prompt": "Реализуй JWT auth middleware",
      "response": "Я создам middleware...",
      "files_touched": ["src/auth.ts"],
      "tools": ["Edit"]
    }
  ],
  "files": [
    { "path": "src/auth.ts", "by_ai": true },
    { "path": "CHANGELOG.md", "by_ai": false }
  ],
  "attribution": {
    "ai_ratio": 73,
    "method": "line",
    "lines": { "ai_added": 146, "total_added": 200, "deleted": 12 }
  }
}
```

<Aside type="tip">
  Списки prompt берутся из причинно связанного диалога вокруг final diff, а не из всей истории сессии. В режиме file-level Agent Note оставляет только путь последнего edit и prompt-only context, который к нему ведёт.
</Aside>

<Aside>
  Распространённые generated artifacts исключаются из знаменателя `AI ratio` в best effort, включая web build outputs, Flutter registrants, Dart codegen, Go protobufs, Rust bindings и generated sources для Swift / Kotlin.
</Aside>

## Жизненный цикл сессии

| Событие | Что происходит |
|---|---|
| **SessionStart** | Создаёт директорию сессии и записывает heartbeat |
| **UserPromptSubmit** | Добавляет prompt, увеличивает turn counter и ротирует logs |
| **PreToolUse Edit/Write** | Синхронно захватывает pre-edit blob hash |
| **PostToolUse Edit/Write** | Захватывает post-edit blob hash и путь к файлу |
| **Stop** | Логирует событие stop (heartbeat остаётся активным — Stop = конец response, а не конец сессии) |

<Aside>
  Git hooks проверяют свежесть сессии по heartbeat перед вставкой trailers или записью notes. Heartbeat обновляется на SessionStart и при каждом prompt.
</Aside>

<Aside type="tip">
  Agent Note знает, какой agent активен. Метаданные сессии хранят текущий adapter, поэтому `show`, `status` и запись commit позже могут определить правильный формат transcript.
</Aside>

## Git Notes

Git notes невидимы в списках веток, в UI GitHub и в CI, но их можно push и fetch.

```bash
# Поделиться с командой
git push origin refs/notes/agentnote

# Забрать из remote
git fetch origin refs/notes/agentnote:refs/notes/agentnote

# Посмотреть note
git notes --ref=agentnote show <commit>
```

<Aside type="tip">
  `agent-note init` настраивает auto-fetch, а hook `pre-push` отвечает за auto-push. Ручные команды не нужны.
</Aside>
