## Что вы получаете

Dashboard даёт команде один общий URL для просмотра истории Agent Note.

- Открытые и уже смерженные PR в одном месте
- Коммиты, prompts и attribution без необходимости открывать каждый PR
- Один URL GitHub Pages, обычно по пути `/dashboard/`

Используйте Dashboard, когда отчёта в PR уже недостаточно и команде нужен общий обзор.

## Перед началом

Dashboard нужен только тогда, когда вам нужен общий веб-интерфейс.

- Если отчётов о сессиях ИИ внутри pull request уже достаточно, остановитесь на [GitHub Action](./github-action/).
- Если вам нужен общий Pages-экран для открытой и смерженной истории, продолжайте здесь.

## Быстрая настройка

1. Run Dashboard setup once.

```bash
npx agent-note init --agent claude --dashboard
```

2. Commit both workflow files.

```bash
git add .github/workflows/agentnote-pr-report.yml .github/workflows/agentnote-dashboard.yml
git commit -m "chore: enable agent-note dashboard"
git push
```

3. In GitHub Pages, choose **GitHub Actions** as the source.

The generated Dashboard workflow restores data from `gh-pages`, refreshes it from `refs/notes/agentnote`, and publishes the shared `/dashboard/` view.

## Публичный URL

Обычно Dashboard публикуется здесь:

- `https://<owner>.github.io/<repo>/dashboard/`

Например:

- `https://wasabeef.github.io/AgentNote/dashboard/`

<Aside type="tip">
  Если вы публикуете через GitHub Pages, workflow Pages передаёт значения `SITE` и `BASE`, зависящие от репозитория, во время build. В большинстве репозиториев можно использовать пример workflow без изменений в `packages/dashboard`.
</Aside>

## Первый deploy

Даже новый репозиторий может накапливать данные Dashboard до первого production-deploy. Публичный URL Pages появится только после первого deploy в `default branch`.

## Как это работает

- `refs/notes/agentnote` remains the source of truth
- The generated Dashboard workflow writes shared snapshots to `gh-pages/dashboard/notes/*.json`
- GitHub Pages serves the built Dashboard at `/dashboard/`

The repository does not commit sample Dashboard JSON. A fresh Dashboard stays empty until the workflow restores or generates note files. `pull_request` refreshes open history, and `default branch` publishes the merged state.
