## Что вы получаете

Dashboard дает команде один общий URL для истории Agent Note.

- История open и merged PR в одном месте
- Просмотр commit, prompt и attribution без открытия каждой PR
- Одна общая URL GitHub Pages, обычно по пути `/dashboard/`

Используйте это, когда PR report уже недостаточно и нужна общая командная страница.

## Перед началом

Dashboard нужен только если вам нужен общий web view.

- Если AI session reports внутри pull request уже достаточно, остановитесь на [GitHub Action](./github-action/).
- Если нужен общий Pages view для истории open и merged, продолжайте здесь.

## Quick Setup

Сначала включите запись Dashboard bundles для каждой PR в Action:

```yaml
- uses: wasabeef/AgentNote@v0
  id: agent-note
  with:
    pr_output: description
    dashboard: true
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Это создаст или обновит `packages/dashboard/public/notes/*.json` для текущей PR. Затем ваш Pages workflow сможет восстановить, сохранить и опубликовать эти файлы.

## Публичный URL

Обычно Dashboard публикуется здесь:

- `https://<owner>.github.io/<repo>/dashboard/`

Например:

- `https://wasabeef.github.io/AgentNote/dashboard/`

## Конфигурация deploy

`packages/dashboard` сохраняет локальный default `/`, поэтому ваш Pages workflow должен подставлять значения, специфичные для repo, во время build:

- `SITE=https://<owner>.github.io`
- `BASE=/<repo>/dashboard`
- `PUBLIC_REPO=<owner>/<repo>`
- `PUBLIC_REPO_URL=https://github.com/<owner>/<repo>` (optional)

Так package остается generic, но при этом генерирует правильные публичные URL для каждого репозитория.

## Рекомендуемый поток

1. Включите `dashboard: true` в GitHub Action.
2. Восстановите `gh-pages/dashboard/notes` в `packages/dashboard/public/notes`.
3. На `pull_request` (`opened`, `reopened`, `synchronize`) переписывайте note files этой PR и сохраняйте их обратно в `gh-pages`.
4. На `push` в `main` помечайте соответствующие PR notes как `merged`, пересобирайте сайт и deploy общий Pages URL.

Так `main` остается чистым от generated JSON, а общий Dashboard все равно показывает историю open и merged PR.

## Первый deploy

Новый repo может накапливать Dashboard data еще до первого production deploy, но публичный Pages URL появится только после первого deploy из `main`.

## Как это работает

- `packages/dashboard/` — это interactive Astro app
- `packages/dashboard/public/notes/*.json` — working bundle, который читается во время build
- `gh-pages/dashboard/notes/*.json` — durable store для общей истории

Репозиторий не содержит sample Dashboard JSON. Новый Dashboard остается пустым, пока ваш workflow не восстановит или не создаст note files.
